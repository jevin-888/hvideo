## 1. 软件概述

### 1.1 软件简介

HSVJEngine是一款专业的音视频表演系统，专为Android 11/12平台设计。该系统提供实时音视频处理、多图层渲染、音频效果联动、网络控制等功能，适用于KTV、演出、展览等专业场景。

### 1.2 主要功能

- **高性能渲染**：基于Vulkan 1.1+图形API，支持硬件加速
- **多图层系统**：支持视频、图片、文本、二维码、特效等多种图层类型
- **音频效果联动**：实时音频分析，驱动视觉效果
- **多协议网络控制**：HTTP、TCP、UDP、WebSocket、RS232/485
- **硬件解码**：支持RKMPP和MediaCodec硬件加速
- **区域矩阵**：支持输入/输出矩阵映射、区域拼接和多屏布局
- **播放列表管理**：支持视频、图片播放列表
- **场景管理**：支持场景切换和配置保存

### 1.3 应用场景

- KTV点歌系统
- 舞台演出视觉效果
- 展览展示系统
- 多媒体互动装置

## 2. 系统架构

### 2.1 整体架构

HSVJEngine采用分层架构设计，从上到下分为：

1. **应用层**：Java代码，提供Android Activity和UI，通过JNI调用C++核心引擎
2. **核心引擎层**：Engine、CommandRouter、SceneManager、LicenseManager等核心组件
3. **功能模块层**：Layer、Decoder、Audio、Effect、Network、Database等功能模块
4. **渲染层**：VulkanRenderer、CaptureRenderer、TextRenderer等渲染器
5. **硬件抽象层**：封装Vulkan、RKMPP、AAudio、V4L2、DRM等底层API

### 3.3 Decoder解码模块

#### 3.3.1 模块特性

- 模块化架构
- 硬件加速（RKMPP、MediaCodec）
- 零拷贝优化
- 帧同步管理
- 对象池优化
- 自动降级机制
- 错误恢复

#### 3.3.2 解码流程

1. 打开视频文件（avformat_open_input）
2. 查找流信息（avformat_find_stream_info）
3. 通过 HardwareManager 选择最佳解码器
4. 配置硬件解码上下文
5. 打开解码器（avcodec_open2）
6. 读取数据包（av_read_frame）
7. 解码帧（avcodec_send_packet/avcodec_receive_frame）
8. 处理零拷贝帧（DRM_PRIME 或 MediaCodec）
9. 帧同步（FrameSyncManager）
10. 放入帧池（FramePool）

#### 3.3.3 零拷贝路径

- AV_PIX_FMT_DRM_PRIME → RKMPP 零拷贝（DMA-BUF）
- AV_PIX_FMT_MEDIACODEC → MediaCodec 零拷贝（AHardwareBuffer）

#### 3.3.4 硬件管理器（HardwareManager）

**解码器优先级策略**：
- RKMPP_FIRST：RK芯片优先RKMPP
- MEDIACODEC_FIRST：其他芯片优先MediaCodec
- SOFTWARE_ONLY：仅软解

**支持的解码器**：
- H.264：h264_rkmpp、h264_mediacodec、h264
- HEVC：hevc_rkmpp、hevc_mediacodec、hevc
- MPEG1：mpeg1_rkmpp、mpeg1video
- MPEG2：mpeg2_rkmpp、mpeg2_mediacodec、mpeg2video
- MJPEG：mjpeg_rkmpp、mjpeg

#### 3.3.5 帧同步管理器（FrameSyncManager）

**同步模式**：
- NORMAL：正常模式（严格同步）
- CATCHUP：追赶模式（宽松同步，用于 seek/resume 后）
- FREE_RUN：自由运行（不同步，用于测试）

**同步阈值**：
- 超前阈值：40ms
- 滞后阈值：300ms
- 追赶模式超前阈值：1s
- 追赶模式滞后阈值：1s

#### 3.3.6 帧对象池（FramePool）

**功能**：
- 帧对象复用减少内存分配（减少 80% 的 new/delete 调用）
- 线程安全
- 统计信息（分配、回收、峰值等）
- 自动扩容

**帧池大小**：
- 默认最大容量：10帧
- 可动态调整

### 3.6 Effect效果模块

#### 3.6.1 EffectManager（效果管理器）

**职责**：
- 音频数据分析
- BPM检测
- 节拍跟踪
- 密集鼓点检测
- 高潮检测

#### 3.6.2 BPM检测

**算法**：峰值形态验证 + 间隔聚类 + 动态平滑

**检测流程**：
1. 能量窗口分析（20帧）
2. 峰值形态验证
3. 记录峰值间隔
4. 间隔聚类统计
5. 计算BPM
6. 动态平滑

**BPM范围**：60-200 BPM

#### 3.6.3 节拍跟踪

**功能**：区分强拍和普通拍

**小节结构**：4/4拍，每小节4拍

**强拍判定**：
- 位置0：强拍（小节第一拍）
- 位置1-3：普通拍

#### 3.6.4 密集鼓点检测

**算法**：SuperFlux Onset Detection（V3.0）

**检测指标**：
- Onset密度
- 高频能量
- 频谱通量
- 响度

**状态机**：
- 待机状态
- 触发状态
- 密集状态
- 退出延迟
- 冷却状态

#### 3.6.5 高潮检测

**算法**：多特征融合

**特征维度**：
- 能量特征（35%权重）
- 频谱熵（25%权重）
- 节拍密度（25%权重）
- 低音占比（15%权重）

**输出**：0-1连续评分

## 6. 算法设计

### 6.1 BPM检测算法

#### 6.1.1 算法概述

采用峰值形态验证 + 间隔聚类 + 动态平滑的方法检测音频BPM。

#### 6.1.2 检测流程

1. **能量窗口分析**：使用20帧滑动窗口计算音频能量
2. **峰值形态验证**：验证峰值是否符合节拍特征
3. **记录峰值间隔**：记录相邻峰值的时间间隔
4. **间隔聚类统计**：对间隔进行聚类，找出主要间隔
5. **计算BPM**：根据主要间隔计算BPM值
6. **动态平滑**：使用移动平均平滑BPM值

#### 6.1.3 算法特点

- BPM范围：60-200 BPM
- 自适应窗口大小
- 抗噪声干扰
- 实时响应

### 6.2 密集鼓点检测算法

#### 6.2.1 算法概述

采用SuperFlux Onset Detection（V3.0）算法检测密集鼓点。

#### 6.2.2 检测指标

- Onset密度：单位时间内的起音数量
- 高频能量：高频段能量占比
- 频谱通量：频谱变化率
- 响度：整体响度水平

#### 6.2.3 状态机

```
待机状态 → 触发状态 → 密集状态 → 退出延迟 → 冷却状态
   ↑                                                    ↓
   └────────────────────────────────────────────────────┘
```

### 6.3 高潮检测算法

#### 6.3.1 算法概述

采用多特征融合的方法检测音乐高潮部分。

#### 6.3.2 特征维度

- **能量特征**（35%权重）：整体能量水平
- **频谱熵**（25%权重）：频谱复杂度
- **节拍密度**（25%权重）：节拍密集程度
- **低音占比**（15%权重）：低频能量占比

#### 6.3.3 输出

- 0-1连续评分
- 实时更新
- 平滑处理

### 6.4 区域矩阵算法

区域矩阵算法负责将输入画布按行列分割，并根据输出矩阵或自由映射关系输出到目标屏幕区域。

#### 6.4.1 区域分割

- 按输入行列计算区域源矩形
- 支持自由矩阵映射到输出单元
- 保持区域亮度、对比度、饱和度参数独立

#### 6.4.2 区域拼接

- 根据输出矩阵计算目标矩形
- 将各区域渲染到对应输出位置
- 保持普通区域矩阵输出链路简洁稳定
