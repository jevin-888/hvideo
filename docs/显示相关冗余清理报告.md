## 开机动画修复记录

发现问题：

- 原始问题：系统启动开机动画只显示左上四分之一。
- 二次复核问题：更换 4K 动画包后，开机动画先居中正常显示数秒，随后变成右下角四分之一。

原因：

- 设备 HDMI 物理输出已经固定为 `3840x2160@30`。
- bootanimation 阶段直接按物理输出显示，原 `/odm/media/bootanimation1.zip` 的 `desc.txt` 为 `1920 1080 20`，因此在 4K 输出上只占 1/4。
- 4K 动画包修复后，又发现 Android 持久化的 `display_size_forced=1920,1080` 会在 bootanimation 仍未退出时生效，导致 4K bootanimation layer 被 1080 逻辑视口裁切。

处理过程：

| 项目 | 结果 |
| --- | --- |
| 原动画包 | 已备份到 `D:\CHUANGWEI\scratch\bootanimation1_current.zip` |
| 设备端原动画备份 | `/data/local/tmp/bootanimation1_before_4kfix.zip` |
| 新动画包 | `D:\CHUANGWEI\scratch\bootanimation1_4k30_fixed.zip` |
| 新动画 desc | `3840 2160 20` |
| 系统安装路径 | `/odm/media/bootanimation1.zip` |
| UI 1080 处理 | 改为开机动画结束后运行时执行 `wm size 1920x1080`，并删除 `display_size_forced` 持久记录 |
| Watchdog APK | 已安装 `com.hsvj.engine` 版本 `1.0.2087-beta-debug`，内置新版 `hsvj_watchdog.sh` |

注意事项：

- `/odm` 分区只有约 6.3 MB，原 `bootanimation2.zip` 占约 5.5 MB，导致首次覆盖 `bootanimation1.zip` 时空间不足，目标文件被截成 0 字节。
- 已将 `/odm/media/bootanimation2.zip`、`bootanimation3.zip`、`bootanimation4.zip` 先备份后从 `/odm/media` 移除，释放空间后再安装 4K 开机动画。
- 不再将 `display_size_forced=1920,1080` 保存在 Settings 中，避免下次开机动画阶段提前套用 1080 逻辑视口。
- `/data/local/tmp/hsvj_watchdog.sh`、`D:\CHUANGWEI\app\src\main\assets\hsvj_watchdog.sh`、`D:\CHUANGWEI\hsvj_watchdog.sh` 已同步为新版逻辑。
- 本地备份：
  - `D:\CHUANGWEI\scratch\bootanimation2_odm_backup.zip`
  - `D:\CHUANGWEI\scratch\bootanimation3_odm_backup.zip`
  - `D:\CHUANGWEI\scratch\bootanimation4_odm_backup.zip`
- 设备端备份：
  - `/data/local/tmp/odm_bootanimation_backup_20260625/bootanimation2.zip`
  - `/data/local/tmp/odm_bootanimation_backup_20260625/bootanimation3.zip`
  - `/data/local/tmp/odm_bootanimation_backup_20260625/bootanimation4.zip`

修复后状态：

