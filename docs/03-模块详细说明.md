## 1. Core 核心模块

### 1.1 模块概述

Core模块是HSVJEngine的核心，负责系统初始化、生命周期管理、命令路由、场景管理和授权验证。

**目录**：`src/core/`、`include/core/`

**主要文件**：
- `Engine.cpp/h`：引擎主类
- `CommandRouter.cpp/h`：命令路由器
- `SceneManager.cpp/h`：场景管理器
- `LicenseManager.cpp/h`：授权管理器
- `SystemConfig.cpp/h`：系统配置
- `Mubu.cpp/h`：图层管理器

### 1.2 Engine（引擎类）

**职责**：
- 系统初始化和关闭
- 模块生命周期管理
- 渲染循环控制
- 资源清理

**关键方法**：
```cpp
bool initialize(ANativeWindow *window);  // 初始化引擎
void shutdown();                         // 关闭引擎
void run();                              // 运行主循环
void update(float deltaTime);            // 更新（每帧）
void renderFrame();                      // 渲染一帧
```

**初始化流程**：
1. 创建系统配置（SystemConfig）
2. 创建授权管理器（LicenseManager）
3. 创建Vulkan渲染器（VulkanRenderer）
4. 创建图层管理器（Mubu）
5. 创建命令路由器（CommandRouter）
6. 创建播放列表管理器（PlaylistManager）
7. 创建场景管理器（SceneManager）
8. 创建效果管理器（EffectManager）
9. 创建音频处理器（AudioProcessor）
10. 初始化网络模块（NetworkManager）

### 1.3 CommandRouter（命令路由器）

**职责**：
- 接收和解析命令请求
- 路由到相应的处理器
- 生成统一的响应格式

**支持的命令类型**（11种）：
- `0x00`：系统配置
- `0x01`：图层管理
- `0x02`：视频播放
- `0x03`：图层渲染
- `0x04`：音频响应
- `0x05`：性能
- `0x06`：同步
- `0x08`：播放列表
- `0x09`：场景
- `0x0A`：特效
- `0x0C`：区域配置
- `0x50/0x51`：中控配置

**命令格式**：
```json
{
  "type": 1,
  "code": 0,
  "requestId": "req-123",
  "timestamp": 1234567890,
  "param": "{...}"
}
```

**响应格式**：
```json
{
  "code": 0,
  "requestId": "req-123",
  "timestamp": 1234567890,
  "traceId": "trace-456",
  "ok": true,
  "error": 0,
  "message": "success",
  "data": "{...}"
}
```

### 1.4 SceneManager（场景管理器）

**职责**：
- 保存和加载场景配置
- 场景切换
- 场景模板管理

**场景配置内容**：
- 图层配置（位置、大小、可见性等）
- 系统配置（分辨率、音量等）
- 效果配置（特效参数）
- 播放列表状态

**使用示例**：
```cpp
// 保存场景
sceneManager.saveScene("scene1", "我的场景");

// 加载场景
sceneManager.loadScene("scene1");

// 获取场景列表
auto scenes = sceneManager.listScenes();
```

### 1.5 LicenseManager（授权管理器）

**职责**：
- 读取和验证授权文件
- 检查功能授权
- 显示授权提示

**授权功能**：
- 图层数量限制
- 功能模块授权
- 有效期验证
- 设备绑定

### 1.6 SystemConfig（系统配置）

**职责**：
- 管理系统配置参数
- 配置持久化
- 配置验证

**配置项**：
- 分辨率设置
- 音量设置
- 网络端口
- 路径配置
- 区域配置

### 1.7 PeripheralManager（中控配置器）

**职责**：
- 外部设备管理（RS232/RS485/DMX512/UDP/TCP/WebSocket）
- 串口通信
- DMX512 控制
- 网络协议管理
- 指令映射和执行

**支持的外设类型**：
- RS232：串口通信
- RS485：工业总线通信
- DMX512：灯光控制协议（串口/ArtNet/sACN）
- UDP：UDP 网络通信
- TCP：TCP 网络通信（服务器/客户端模式）
- WebSocket：WebSocket 通信

**DMX512 功能**：
- 串行 DMX512（250kbps）
- ArtNet 协议
- sACN 协议
- 实时通道控制
- 模拟调试模式

**关键方法**：
```cpp

void setSystemConfig(SystemConfig* config);
void setEngine(Engine* engine);
void setCommandRouter(CommandRouter* router);
void broadcastResult(const std::string& resultJson);
Json::Value processPeripheralCommand(const std::string& action, const Json::Value& param);
void setDmxMaster(int value);

uint8_t getChannelValue(int channel) const;
```

## 3. Decoder 解码模块

### 3.1 模块概述

Decoder模块负责视频文件解码和V4L2设备采集，支持硬件加速解码。

**目录**：`src/decoder/`、`include/decoder/`

**核心特性**：
- 模块化架构
- 硬件加速（RKMPP、MediaCodec）
- 零拷贝优化
- 帧同步管理
- 对象池优化
- 自动降级机制
- 错误恢复

### 3.2 VideoDecoder（解码器主接口）

**职责**：
- 视频文件打开/关闭
- 播放控制
- 帧获取
- 参数设置

**关键方法**：
```cpp
bool open(const std::string &path, bool fastOpen = false);
bool openV4L2Device(const std::string &devicePath, ...);
void close();

bool start(double syncStartTime = 0.0);
void pause();
void resume();
void stop();
bool seek(double position);

FramePtr getCurrentFramePtr();  // 推荐
DecodedFrame* getCurrentFrame(); // 需手动释放
void releaseFrame(DecodedFrame *frame);

double getCurrentPosition() const;
double getDuration() const;
int getWidth() const;
int getHeight() const;
```

### 3.3 DecoderCore（解码核心）

**职责**：
- FFmpeg 解码流程封装
- 硬件解码零拷贝支持（RKMPP DRM_PRIME / MediaCodec HWBuffer）
- 音视频同步解码
- 解码线程管理
- 帧生命周期管理（配合 FramePool 和引用计数）
- V4L2 采集设备支持（HDMI RX、USB 摄像头）
- 音频播放器集成
- 音频重采样（SwrContext）
- 音频数据回调（用于音频效果联动）

**解码流程**：
1. 打开视频文件（avformat_open_input）
2. 查找流信息（avformat_find_stream_info）
3. 通过 HardwareManager 选择最佳解码器
4. 配置硬件解码上下文
5. 打开解码器（avcodec_open2）
6. 读取数据包（av_read_frame）
7. 解码帧（avcodec_send_packet/avcodec_receive_frame）
8. 处理零拷贝帧（DRM_PRIME 或 MediaCodec）
9. 帧同步（FrameSyncManager）
10. 放入帧池（FramePool）

**零拷贝路径**：
- AV_PIX_FMT_DRM_PRIME → RKMPP 零拷贝（DMA-BUF）
- AV_PIX_FMT_MEDIACODEC → MediaCodec 零拷贝（AHardwareBuffer）

### 3.4 HardwareManager（硬件管理器）

**职责**：
- 自动检测可用硬件解码器
- 智能选择最佳解码器（根据设备类型）
- 管理硬件设备上下文
- 配置零拷贝模式
- 自动降级机制（硬解失败回退软解）

**解码器优先级策略**：
- RKMPP_FIRST：RK芯片优先RKMPP
- MEDIACODEC_FIRST：其他芯片优先MediaCodec
- SOFTWARE_ONLY：仅软解

**支持的解码器**：
- H.264：h264_rkmpp、h264_mediacodec、h264
- HEVC：hevc_rkmpp、hevc_mediacodec、hevc
- MPEG1：mpeg1_rkmpp、mpeg1video
- MPEG2：mpeg2_rkmpp、mpeg2_mediacodec、mpeg2video
- MJPEG：mjpeg_rkmpp、mjpeg

**关键方法**：
```cpp
bool initialize();
const AVCodec* selectBestDecoder(AVCodecID codecId);
bool configureHardwareDecoding(const AVCodec* codec, AVCodecContext* codecCtx);
ZeroCopyMode configureZeroCopy(const AVCodec* codec, AVCodecContext* codecCtx);
void markDecoderFailed(const std::string& decoderName);
```

### 3.5 FrameSyncManager（帧同步管理器）

**职责**：
- 统一的帧同步逻辑（正常模式、追赶模式、自由运行）
- 精确的同步决策（显示/等待/丢弃）
- 同步统计（丢帧率、平均时间差等）
- 多视频模式支持
- 线程安全

**同步模式**：
- NORMAL：正常模式（严格同步）
- CATCHUP：追赶模式（宽松同步，用于 seek/resume 后）
- FREE_RUN：自由运行（不同步，用于测试）

**同步阈值**：
- 超前阈值：40ms
- 滞后阈值：300ms
- 追赶模式超前阈值：1s
- 追赶模式滞后阈值：1s

**关键方法**：
```cpp
SyncDecision checkFrameSync(double framePts, double currentPlayTime,
                            float playbackRate, SyncMode mode);
void enterCatchupMode();
void exitCatchupMode();
void setMultiVideoMode(bool enabled);
SyncStats getStats() const;
```

### 3.6 FramePool（帧对象池）

**职责**：
- 帧对象复用减少内存分配（减少 80% 的 new/delete 调用）
- 线程安全
- 统计信息（分配、回收、峰值等）
- 自动扩容

**帧池大小**：
- 默认最大容量：10帧
- 可动态调整

**关键方法**：
```cpp
DecodedFrame* acquire();
void recycle(DecodedFrame* frame);
void clear();
Stats getStats() const;
void resize(size_t newMaxSize);
```

### 3.7 DecodedFrame（解码帧对象）

**职责**：
- 自动引用计数（防止内存泄漏和重复释放）
- 支持多种帧类型（软件帧、RKMPP DRM_PRIME、MediaCodec HWBuffer）
- 线程安全
- 对象池友好

**帧类型**：
- SOFTWARE：软件帧（NV12/YUV420P）
- RKMPP_DRM_PRIME：RKMPP 零拷贝（DMA-BUF）
- MEDIACODEC_HWBUFFER：MediaCodec 零拷贝（AHardwareBuffer）

**DRM_PRIME 数据结构**：
```cpp
struct DrmPrimeData {
    int fd;                    // DMA-BUF 文件描述符
    uint32_t width, height;    // 帧尺寸
    uint32_t format;           // DRM 像素格式
    uint64_t modifier;         // DRM 修饰符
    int numPlanes;             // 平面数量
    struct Plane {
        uint32_t offset;       // 平面偏移
        uint32_t pitch;        // 行距
    } planes[4];
};
```

**FramePtr 智能指针**：
- RAII 风格的帧引用管理
- 自动调用 addRef() 和 release()
- 类似 std::shared_ptr

### 3.8 PlaybackStateMachine（播放状态机）

**职责**：
- 严格的状态转换控制
- 状态变更回调
- 线程安全
- 防止非法操作

**播放状态**：
- IDLE：空闲（未打开文件）
- OPENED：已打开文件
- PLAYING：播放中
- PAUSED：已暂停
- SEEKING：跳转中
- STOPPED：已停止
- ERROR：错误状态

**状态转换事件**：
- OPEN、START、PAUSE、RESUME、SEEK、SEEK_COMPLETED、STOP、CLOSE、ERROR_OCCURRED

### 3.9 ErrorHandler（错误处理器）

**职责**：
- 统一的错误处理逻辑
- 自动重试策略
- 硬件解码失败自动回退软解
- 错误统计

**错误类型**：
- OPEN_FILE_FAILED：打开文件失败
- FIND_STREAM_FAILED：查找流失败
- OPEN_CODEC_FAILED：打开解码器失败
- SEND_PACKET_FAILED：发送数据包失败
- RECEIVE_FRAME_FAILED：接收帧失败
- HARDWARE_ERROR：硬件解码错误
- SEEK_FAILED：跳转失败
- RESOURCE_ERROR：资源错误

### 3.10 HardwareBufferHelper（硬件缓冲区辅助类）

**职责**：
- AImageReader + AHardwareBuffer 零拷贝
- 创建专用 Surface 给 MediaCodec
- 从 AImageReader 获取 AHardwareBuffer
- 零拷贝到 Vulkan

**工作流程**：
1. 创建 AImageReader（指定尺寸和格式）
2. 获取 AImageReader 的 ANativeWindow（用于 MediaCodec Surface）
3. MediaCodec 解码输出到此 Surface
4. 从 AImageReader 获取 AImage → AHardwareBuffer
5. AHardwareBuffer 零拷贝导入 Vulkan 纹理

**关键方法**：
```cpp
bool initialize(int width, int height, int format, int maxImages);
ANativeWindow* getSurface() const;
bool acquireLatestFrame(HardwareBufferFrame* outFrame);
void releaseFrame(HardwareBufferFrame* frame);
```

### 3.11 MediaCodecHelper（MediaCodec辅助类）

**职责**：
- MediaCodec AHardwareBuffer 零拷贝

- JNI 环境管理

**关键方法**：
```cpp
bool init(JNIEnv* env);
AHardwareBuffer* getHardwareBuffer(AVFrame* frame);
void releaseHardwareBuffer(AHardwareBuffer* buffer);
static bool getBufferInfo(AHardwareBuffer* buffer, ...);
```

**主要类**：
- `VulkanRenderer`：主渲染器
- `CaptureRenderer`：采集渲染器
- `RegionRotationRenderer`：区域矩阵渲染器

### 4.2 VulkanRenderer（主渲染器）

**关键方法**：

bool initialize(ANativeWindow *window);
void shutdown();
void beginFrame();

void renderLayer(Layer *layer);

### 4.6 RKDRMRenderer（Rockchip DRM渲染器）

**功能**：
- Rockchip DRM AFBC 零拷贝渲染
- 使用 RK 原生 DRM 扩展
- RKMPP AFBC 帧的零拷贝渲染
- DRM 原子操作

**工作流程**：
1. 打开 DRM 设备，获取 CRTC、Connector、Plane
2. 从 RKMPP 获取 AFBC DMA-BUF fd
3. 导入 DMA-BUF 为 GEM，创建 FB
4. 使用 DRM 原子操作将 FB 渲染到 overlay plane
5. 与 Vulkan 渲染的 primary plane 合成

**关键方法**：
```cpp
bool initialize();
bool renderDrmPrimeFrame(const AVFrame* frame, int planeIndex,
                         int dstX, int dstY, int dstW, int dstH);
bool commit();
int getOverlayPlaneCount() const;
```

### 4.7 CaptureRenderer（采集渲染器）

**功能**：
- V4L2 采集设备渲染

## 6. Effect 效果模块

### 6.1 模块概述

Effect模块负责音频效果分析，包括节拍检测、密集鼓点检测和高潮检测。

**目录**：`src/effect/`、`include/effect/`

**主要类**：
- `EffectManager`：效果管理器
- `WaveBeatDetector`：节拍检测器
- `DiscogsDetector`：音乐风格检测器

### 6.2 EffectManager（效果管理器）

**职责**：
- 音频数据分析
- BPM检测
- 节拍跟踪
- 密集鼓点检测
- 高潮检测

**关键方法**：
```cpp

BeatType recordBeatAndGetType(int64_t currentTimeMs, float bpm);
bool isDenseDrumbeat();
bool isClimax() const;
float getClimaxScore() const;
```

### 6.3 BPM检测

**算法**：峰值形态验证 + 间隔聚类 + 动态平滑

**检测流程**：
1. 能量窗口分析（20帧）
2. 峰值形态验证
3. 记录峰值间隔
4. 间隔聚类统计
5. 计算BPM
6. 动态平滑

**BPM范围**：60-200 BPM

### 6.4 节拍跟踪

**功能**：区分强拍和普通拍

**小节结构**：4/4拍，每小节4拍

**强拍判定**：
- 位置0：强拍（小节第一拍）
- 位置1-3：普通拍

**相位校准**：支持0-3的相位偏移

### 6.5 密集鼓点检测

**算法**：SuperFlux Onset Detection（V3.0）

**检测指标**：
- Onset密度
- 高频能量
- 频谱通量
- 响度

**状态机**：
- 待机状态
- 触发状态
- 密集状态
- 退出延迟
- 冷却状态

### 6.6 高潮检测

**算法**：多特征融合

**特征维度**：
- 能量特征（35%权重）
- 频谱熵（25%权重）
- 节拍密度（25%权重）
- 低音占比（15%权重）

**输出**：0-1连续评分
