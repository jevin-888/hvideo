# 分层渲染架构优化方案

## 一、问题分析

### 当前架构缺陷
- **全有或全无**：只要有一个动态元素（视频/采集），所有图层都重绘
- **无法区分更新频率**：静态背景和60fps视频同等对待
- **带宽浪费**：1920×1080@RGBA = 8MB/帧，60fps = 480MB/s

### 典型场景分析
```
Layer 1: 视频 (60fps更新) ← 高频
Layer 10: 采集 (30-60fps)  ← 高频
Layer 21: 歌词 (30fps)     ← 中频
Layer 30: 背景图 (静态)    ← 低频/静态
Layer 40: 字幕 (变化时)    ← 低频
Layer 71: 二维码 (静态)    ← 静态
```

当前：只要Layer 1在播放，**所有6个图层每帧都重绘**
优化后：只重绘Layer 1，其他复用缓存

## 二、分层渲染方案

### 架构设计

```
┌─────────────────────────────────────────┐
│         最终输出 (Swapchain)             │
└─────────────────────────────────────────┘
                   ▲
                   │ 合成
         ┌─────────┴─────────┐
         │                   │
    ┌────▼────┐      ┌───────▼──────┐
    │静态层    │      │   动态层      │
    │缓存      │      │  (每帧重绘)   │
    └─────────┘      └──────────────┘
         │                   │
    仅变化时更新         每帧更新
```

### 图层分类策略

#### 静态层 (Static Layer)
- 条件：无视频、无采集、无动画、无漫游
- 更新：仅在属性变化时（位置、透明度、纹理切换）
- 典型图层：背景图(30)、Logo(70)、静态二维码(71)
- 渲染频率：按需（<1 fps）

#### 低频层 (Low-Frequency Layer) 
- 条件：更新频率 < 30fps
- 更新：定期或条件触发
- 典型图层：歌词(21)、普通字幕(40)
- 渲染频率：10-30 fps

#### 高频层 (High-Frequency Layer)
- 条件：持续更新的动态内容
- 更新：每帧
- 典型图层：视频(1-9)、采集(10-11)
- 渲染频率：60 fps

### 实现步骤

#### 阶段1：引入图层分类
```cpp
// include/core/LayerRenderFrequency.h
enum class LayerRenderFrequency {
    STATIC,      // 静态：仅属性变化时更新
    LOW,         // 低频：10-30fps
    HIGH         // 高频：每帧
};

class LayerFrequencyClassifier {
public:
    static LayerRenderFrequency classify(const Layer* layer) {
        if (!layer) return LayerRenderFrequency::STATIC;
        
        LayerType type = layer->getType();
        
        // 视频图层：检查播放状态
        if (type == LayerType::VIDEO) {
            auto* videoLayer = static_cast<const LayerVideo*>(layer);
            if (videoLayer->isCaptureMode() || 
                videoLayer->getState() == LayerVideo::PlayState::PLAYING) {
                return LayerRenderFrequency::HIGH;
            }
        }
        
        // 歌词图层：中频更新
        if (layer->getLayerId() == 21) {
            return LayerRenderFrequency::LOW;
        }
        
        // 动画图片：高频
        if (type == LayerType::IMAGE) {
            auto* imageLayer = static_cast<const LayerImage*>(layer);
            if (imageLayer->isAnimated()) {
                return LayerRenderFrequency::HIGH;
            }
        }
        
        // 漫游切片：高频
        Json::Value slices;
        if (layer->getAllSlices(slices) && hasActiveSliceRoam(slices)) {
            return LayerRenderFrequency::HIGH;
        }
        
        return LayerRenderFrequency::STATIC;
    }
};
```

#### 阶段2：多缓冲管理
```cpp
// include/renderer/LayeredCanvasManager.h
class LayeredCanvasManager {
public:
    struct LayerGroup {
        LayerRenderFrequency frequency;
        std::vector<std::shared_ptr<Layer>> layers;
        VkImage canvasBuffer;
        VkImageView canvasView;
        VkFramebuffer framebuffer;
        uint64_t contentSignature = 0;
        bool needsRedraw = true;
        int framesSinceUpdate = 0;
    };
    
    void updateGrouping(const std::vector<std::shared_ptr<Layer>>& visibleLayers);
    void renderGroups(VulkanRenderer* renderer);
    void compositeToSwapchain(VulkanRenderer* renderer);
    
private:
    LayerGroup staticGroup_;    // 静态图层组
    LayerGroup lowFreqGroup_;   // 低频图层组  
    LayerGroup highFreqGroup_;  // 高频图层组
};
```

#### 阶段3：智能合成
```cpp
void LayeredCanvasManager::renderGroups(VulkanRenderer* renderer) {
    // 1. 静态层：仅签名变化时重绘
    if (staticGroup_.needsRedraw) {
        renderGroupToBuffer(staticGroup_, renderer);
        staticGroup_.needsRedraw = false;
    }
    
    // 2. 低频层：每N帧或内容变化时重绘
    bool lowFreqNeedsUpdate = false;
    if (lowFreqGroup_.framesSinceUpdate >= 2) {  // 30fps at 60Hz
        lowFreqNeedsUpdate = true;
    }
    for (auto& layer : lowFreqGroup_.layers) {
        if (layer->needsTextureUpdate()) {
            lowFreqNeedsUpdate = true;
            break;
        }
    }
    if (lowFreqNeedsUpdate) {
        renderGroupToBuffer(lowFreqGroup_, renderer);
        lowFreqGroup_.framesSinceUpdate = 0;
    } else {
        lowFreqGroup_.framesSinceUpdate++;
    }
    
    // 3. 高频层：每帧必绘
    renderGroupToBuffer(highFreqGroup_, renderer);
    
    // 4. 合成三层到swapchain（单个draw call，三张纹理混合）
    compositeToSwapchain(renderer);
}

void LayeredCanvasManager::compositeToSwapchain(VulkanRenderer* renderer) {
    // 使用自定义shader一次性合成三层
    // Fragment shader: output = static * alpha_s + lowFreq * alpha_l + highFreq * alpha_h
    renderer->beginSwapchainPass();
    renderer->bindCompositeShader();
    renderer->bindTexture(0, staticGroup_.canvasView);
    renderer->bindTexture(1, lowFreqGroup_.canvasView);
    renderer->bindTexture(2, highFreqGroup_.canvasView);
    renderer->drawFullscreenQuad();
    renderer->endSwapchainPass();
}
```

### 预期收益

#### 典型场景：1个视频 + 歌词 + 3个静态图层
**当前**：
- 5个图层 × 60fps = 300次渲染/秒
- Canvas写入：8MB × 60 = 480MB/s

**优化后**：
- 静态层：5次渲染/秒（仅属性变化时）
- 低频层：30次渲染/秒（歌词）
- 高频层：60次渲染/秒（视频）
- 总渲染：95次/秒 ← **节省68%**
- Canvas写入：~200MB/s ← **节省58%**

#### GPU负载降低
- Draw call减少：从每帧5次 → 每帧1次（高频）+ 每2帧1次（低频）
- Tile memory带宽：降低50-70%
- 功耗降低：估计20-30%

## 三、实施路线

### Phase 1：最小可行方案（1-2周）
1. 添加图层频率分类器
2. 将Layer 71（二维码）单独渲染 ← **已实现**
3. 扩展到所有静态图层

### Phase 2：三层架构（2-3周）
1. 实现静态/低频/高频三层缓冲
2. 实现智能合成shader
3. 性能测试和调优

### Phase 3：精细化优化（1-2周）
1. 自适应频率调整（根据CPU/GPU负载）
2. 切片级别的分层（同一图层的不同切片可在不同层）
3. 预测式缓存失效

## 四、兼容性考虑

### 向后兼容
- 保留现有的 `canvasNeedsRedraw` 机制作为fallback
- 新架构通过配置开关启用
- 逐步迁移，不破坏现有功能

### 配置接口
```json
{
  "renderer": {
    "layered_rendering": {
      "enabled": true,
      "static_layer_enabled": true,
      "low_freq_interval": 2,  // 每2帧更新一次
      "auto_classify": true     // 自动分类图层频率
    }
  }
}
```

## 五、风险评估

### 技术风险
- **中等**：需要修改核心渲染循环
- 缓解：渐进式重构，保持fallback路径

### 内存开销
- **低**：额外2-3个framebuffer（~24-36MB）
- 相对于节省的带宽（数百MB/s），可接受

### 测试复杂度
- **中等**：需要覆盖各种图层组合
- 缓解：单元测试 + 性能基准测试

## 六、替代方案对比

| 方案 | 性能提升 | 实现复杂度 | 兼容性 | 推荐度 |
|------|---------|-----------|--------|--------|
| 分层渲染 | ★★★★★ | ★★★ | ★★★★ | ★★★★★ |
| 脏矩形 | ★★★ | ★★★★★ | ★★★ | ★★ |
| 降低更新率 | ★★ | ★ | ★★★★★ | ★★★ |
| GPU Instancing | ★★★ | ★★ | ★★★★★ | ★★★★ |

### 为什么选择分层渲染？
1. **性能提升最大**：50-70%的渲染负载降低
2. **架构清晰**：符合现有设计模式
3. **可渐进实施**：不需要一次性重写
4. **用户透明**：不影响现有功能和配置
