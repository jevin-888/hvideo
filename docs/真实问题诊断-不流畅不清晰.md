## 🔍 真实问题诊断

### 问题1：实际帧率只有24fps（不流畅）

```
[RenderLoopStats] actualFps=24.2
avgUpdateMs=40.1ms  ← 每帧耗时40ms
maxUpdateMs=53ms    ← 最慢53ms
```

**分析**：
- 目标：60fps（16.67ms/帧）
- 实际：24fps（40ms/帧）
- **帧率不足60%**，用户感觉明显卡顿

### 问题2：超4K视频重新播放了！

```
[SwitchStall] layer=1 stage=decoder_open
path=/huoshan/video/人像视频2625x2160.mp4
size=2626x2160  ← 超4K分辨率！
```

**关键发现**：
- ✅ Layer 10（采集）：1920×1080 - 正常
- ❌ **Layer 1（视频）：2626×2160 - 超4K！**
- ❌ Layer 41（文字）：800×200 - 正常

**这就是卡顿的根本原因！**

### 问题3：显示分辨率缩放（不清晰）

从API返回的配置看：
```json
Layer 10 (采集):
  size: 1413x2032  ← 显示尺寸
  tex: 1920x1080   ← 实际采集分辨率
  after: 584.5,0.0 751.0x1080.0  ← 渲染后尺寸（被裁剪！）
```

**分析**：
- 采集源：1920×1080
- 显示区域：1413×2032（竖屏）
- **实际渲染：751×1080（被严重裁剪！）**
- **只显示了原始采集的39%内容**（751/1920 = 39%）

**这就是不清晰的原因：**
1. 1920×1080的采集被压缩到751×1080
2. 损失了60%的横向分辨率
3. fitMode=1（保持比例）导致大量内容被裁剪

## 🎯 根本原因总结

### 原因1：超4K视频拖累性能 ⭐⭐⭐⭐⭐

**数据支持**：
```
当前配置：2626×2160视频 + 1920×1080采集
实际性能：24fps，40ms/帧
引擎CPU：35.2%
状态：❌ 卡顿
```

**对比之前测试**：
```
仅采集配置：1920×1080采集
实际性能：60fps+（推测）
引擎CPU：13.7%
状态：✅ 流畅
```

**结论**：超4K视频导致：
- 帧时间增加 2.5倍（16ms → 40ms）
- 帧率降低 60%（60fps → 24fps）
- CPU增加 157%（13.7% → 35.2%）

### 原因2：采集图层尺寸配置不当 ⭐⭐⭐⭐

**当前配置问题**：
```
显示区域：1413×2032（竖屏，比例约0.7）
采集源：1920×1080（横屏，比例约1.78）
fitMode=1（保持比例）

结果：只能显示采集源的中间39%
```

**不清晰的原因**：
1. 采集1920×1080被压缩到751宽度
2. 两侧被裁剪（1920-751=1169像素丢失）
3. 实际显示分辨率严重降低

## 🚀 立即解决方案

### 方案1：停止播放超4K视频（紧急！）

**通过Web界面**：
- 打开 http://192.168.1.100
- 找到Layer 1（视频图层）
- 点击"隐藏"或"删除"

**或通过API**：
```bash
# 隐藏Layer 1
curl -X POST http://192.168.1.100/api/layer/1/visible \
  -H "Content-Type: application/json" \
  -d '{"visible": false}'
```

**预期效果**：
- 帧率：24fps → 60fps
- 帧时间：40ms → <16ms
- CPU：35% → 15%
- 流畅度：❌ → ✅

### 方案2：调整采集图层尺寸和适配模式（修复清晰度）

**问题分析**：
- 当前：1413×2032竖屏区域 + 1920×1080横屏采集 = 严重裁剪
- 目标：充分显示采集内容，避免裁剪

**解决方案A：使用1920×1080横屏显示区域**
```bash
curl -X POST http://192.168.1.100/api/layer/10/size \
  -H "Content-Type: application/json" \
  -d '{"width": 1920, "height": 1080}'

curl -X POST http://192.168.1.100/api/layer/10/position \
  -H "Content-Type: application/json" \
  -d '{"x": 0, "y": 0}'
```

**解决方案B：改用fitMode=0（拉伸填充）**
```bash
curl -X POST http://192.168.1.100/api/layer/10/fitMode \
  -H "Content-Type: application/json" \
  -d '{"fitMode": 0}'
```

**预期效果**：
- 显示完整采集画面
- 分辨率：751×1080 → 1920×1080（提升156%）
- 清晰度：❌ → ✅

### 方案3：如果需要视频，使用1080p版本

**当前视频**：2626×2160 (22.7MB/帧)
**推荐使用**：1920×1080 (8.3MB/帧)

**好处**：
- 减少内存占用63%
- 减少GPU带宽63%
- 预期帧率：40-50fps
- 仍可流畅播放
