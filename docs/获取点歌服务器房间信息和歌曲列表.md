客户端状态获取与实时更新 — 完整接口文档
一、获取房间状态（HTTP 轮询 / 初始化）
GET /api/v1/rooms/:id/state
获取房间当前的播放状态、音量、外设状态等所有信息。

认证: 无需

房间ID支持4种格式:

current — 最常用，服务器自动根据请求来源IP识别房间
192.168.1.100 — 直接传终端IP
A01 — 房间名称
550e8400-... — 房间UUID
示例请求:

GET /api/v1/rooms/current/state //示例

字段说明:

字段	类型	说明
id	string	房间UUID
name	string	房间名称，如 "A01"
status	int	房间状态：0=空闲 1=使用中 2=维护中

current_song_title	string|null	当前播放歌曲名
ac_state	object	空调状态（见下）
light_state	object	灯光状态（见下）
effect_state	object	音效状态（见下）
room_ip	string|null	绑定终端的IP地址

ac_state:

wind	string	风速：auto/low/mid/high
light_state:

字段	类型	说明
scene	string	灯光场景：auto/happy/romantic/off 等
effect_state:

字段	类型	说明
mode	string	音效模式：standard/KTV/concert 等

三、实时状态推送（WebSocket）
这是状态变化通知的核心机制，所有状态变更都通过 WebSocket 推送，不需要轮询。

连接方式
ws://服务器IP:端口/ws?roomId=current
连接时带上 roomId 参数，服务器会立即主动推送完整初始状态，无需额外请求。

连接后服务器自动推送的消息
消息1 — 初始状态 (initialState):

{
  "type": "initialState",
  "data": {
    "room_id": "room-uuid",
    "room_name": "A01",
    "status": 1,
    "play_state": 1,
    "volume": 50,
    "music_volume": 50,
    "mic_volume": 30,
    "mute": false,
    "mic_status": 1,
    "current_song_id": "song-uuid",
    "current_song_title": "月亮代表我的心",
    "ac": { "power": true, "temp": 26, "mode": "auto" },
    "light": { "scene": "auto" },
    "effect": { "mode": "standard" }
  }
}
消息2 — 初始播放列表 (getPlayList):

{
  "type": "getPlayList",
  "data": [ /* 同上面队列项结构 */ ],
  "listType": 1
}
后续实时推送的消息类型
type 值	触发时机	说明
room_update / roomStateChanged	房间任何状态变更	包含完整房间状态
getPlayList	队列变化（点歌/切歌/清空/置顶）	包含完整队列数组
autoNotifyMusBarState / getMusBarStateResponse	音量/麦克风/播放状态变化	音控栏状态
command	服务器下发控制指令	播放控制命令
terminal_status	终端上下线	终端在线状态变化
room_update 消息示例:

{
  "type": "room_update",
  "data": {
    "id": "room-uuid",
    "play_state": 2,
    "volume": 60,
    "mute_status": 0,
    ...
  }
}
前端监听示例
// 连接时带 roomId，服务器自动推送初始状态
const ws = new WebSocket('ws://192.168.1.1:8080/ws?roomId=current');

ws.onmessage = (event) => {
    const msg = JSON.parse(event.data);

    switch (msg.type) {
        case 'initialState':
            // 初始化UI状态
            updateRoomState(msg.data);
            break;

        case 'getPlayList':
            // 更新播放列表
            updatePlaylist(msg.data);
            break;

        case 'room_update':
        case 'roomStateChanged':
            // 房间状态变化（音量、播放状态等）
            updateRoomState(msg.data);
            break;

        case 'autoNotifyMusBarState':
        case 'getMusBarStateResponse':
            // 音控栏状态更新
            updateMusicBar(msg.data);
            break;
    }
};
四、推荐的客户端初始化流程
1. 建立 WebSocket 连接（带 roomId=current）
   ↓
2. 等待 initialState 消息 → 初始化房间状态UI
   ↓
3. 等待 getPlayList 消息 → 初始化播放列表UI
   ↓
4. 后续所有变化通过 WebSocket 实时接收，无需轮询
如果需要主动刷新（如 WebSocket 断线重连后），可以直接调用 HTTP 接口：

GET /api/v1/rooms/current/state — 刷新房间状态
GET /api/v1/rooms/current/queue — 刷新播放列表
