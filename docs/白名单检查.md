# HSVJ 引擎资产安全与性能优化部署指南 (2026-04-02)

## 0. 核心背景
HSVJ 引擎工业版对当前数据根目录 `ROOT_PATH`（优先 `/huoshan/`，找不到时才 `/sdcard/huoshan/`）有严格的 **“正身核查”** 与 **“安全准入”** 机制。任何不在白名单中的文件或目录，在引擎启动时都会被 **自动清理 (Cleanup)**。

为了实现 4K MIPI 采集优化及 OOM 护体 (-1000 优先级)，应用会从 APK assets 直接把 `hsvj_optimize.sh` 等脚本复制到应用私有目录后执行。下面的白名单链路只适用于需要同步到数据根目录 `ROOT_PATH` 的非音视频资产。

---

## 1. 资产部署链路 (五步法)

如果你以后要增加新的非音视频资产 (如新模型、私有配置)，**必须** 在以下 5 个位置登记：

### 第 1 步：APK 资产预装
- **路径**: `app/src/main/assets/`
- **操作**: 将源文件放入此目录 (如 `assets/scripts/hsvj_optimize.sh`)。这样 Gradle 才会把它打包进 APK。

### 第 2 步：Java 层资源提取白名单
- **文件**: `app/src/main/java/com/hsvj/engine/StartupConfig.java`
- **位置**: `CRITICAL_ASSET_DIRS` 数组。
- **作用**: Java 启动时会遍历此列表，决定哪些 assets 目录需要提取到磁盘。

### 第 3 步：Java 层资产完整性强制校验
- **文件**: `app/src/main/java/com/hsvj/engine/resource/EmbeddedResourceManager.java`
- **函数**: `areCriticalResourcesExtracted()`。
- **作用**: 对启动必需资源做强制校验，防止增量更新或磁盘误删后继续启动。

### 第 4 步：C++ 目录合法性白名单 (Web 权限)
- **文件**: `src/network/HttpServer_Helpers.cpp`
- **位置**: `isPathAllowed` 函数中的 `subdirs` 数组。
- **作用**: 如果 Web 管理界面或 API 需要访问此目录，必须在此备案，否则会返回 `403 Access Denied`。

### 第 5 步：C++ 引擎自清理白名单 (最重要)
- **文件**: `src/core/Engine_Init.cpp`
- **位置**: `cleanupUnnecessaryFiles` 函数中的 `requiredDirs` 数组。
- **作用**: **这是防止文件被自动删除的关键。** 引擎启动会遍历 ROOT_PATH 下所有子目录，不在该列表中的目录将被彻底删除。

---

## 2. 性能优化脚本专有配置

### 2.1 路径
脚本不进入数据根目录，也不依赖 `ROOT_PATH`。`MainActivity.copyScriptFromAssets()` 会从 `app/src/main/assets/` 复制到应用私有目录后执行。

### 2.2 自动执行逻辑
执行代码位于 `src/core/Engine.cpp` 的 `initialize()` 后段：
- **执行时机**: 在 DMX 及网络初始化完成后。
- **执行方式**:
  - 采用全兼容 `su` 语法 (适配不支持 `-c` 参数的工业固件)。
  - 强制进程变更为 `-1000` 优先级。
  - 关闭系统静默干扰、禁用无关后台服务。

---

## 3. 常见问题排查 (Troubleshooting)

| 故障现象 | 可能原因 | 解决办法 |
| :--- | :--- | :--- |
| **脚本执行了但不生效 (-1000 没变)** | `su` 权限被厂商锁定或 shell 语法不兼容 | 检查 Engine.cpp 中 `su` 命令的各种 fallback 变体是否覆盖 |
| **脚本在磁盘上找不到了** | APK assets 缺失或私有目录复制失败 | 确认脚本在 `app/src/main/assets/`，并检查 `MainActivity.copyScriptFromAssets()` 日志 |
| **API 调用资源返回 403** | 路径白名单未包含 | 检查 `HttpServer_Helpers.cpp` 的 `subdirs` |
| **Logcat 报错找不到启动资源** | Java 资源同步失败 | 检查 `EmbeddedResourceManager` 与 `IncrementalUpdater` 日志 |
