# 授权工具 — 云端认证与授权接口设计

本文档描述代理商授权 exe 与云端后台之间的**认证**与**授权相关接口**，包括登录、Token 使用、获取名下设备、以及可选签发记录上报。

---

## 1. 认证概述

- **角色**：代理商（使用授权 exe 签发 license.dat）。
- **认证方式**：账号 + 密码登录，服务端返回 **Access Token**；后续所有授权相关接口均在请求头中携带 Token，服务端按 Token 识别代理商身份并做权限与数据范围校验。
- **安全要求**：HTTPS；Token 具备有效期，支持刷新；exe 本地存储 Token 时建议加密或使用系统凭据。

---

## 2. 通用约定

### 2.1 请求基础
ssh root@60.205.127.117
- **Base URL**：由云端后台提供，例如 `https://api.example.com` 或 `https://license-api.example.com`。
- **Content-Type**：`application/json`（除登录外若需表单则另行说明）。
- **字符编码**：UTF-8。

### 2.2 需登录接口的请求头

除登录、刷新 Token 外，其余接口需在请求头中携带 Token：

```
Authorization: Bearer <access_token>
```

或（若后端约定使用自定义头）：

```
X-Access-Token: <access_token>
```

### 2.3 统一响应结构

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 业务码，0 表示成功 |
| message | string | 提示信息，可选 |
| data | object / array | 业务数据，可选 |

示例：

```json
{
  "code": 0,
  "message": "success",
  "data": { }
}
```

失败示例：

```json
{
  "code": 401,
  "message": "未登录或 Token 已失效",
  "data": null
}
```

### 2.4 常见业务码

| code | 含义 |
|------|------|
| 0 | 成功 |
| 401 | 未登录、Token 缺失或已失效 |
| 403 | 无权限（如设备不属于当前代理商） |
| 400 | 参数错误 |
| 500 | 服务端错误 |

## 3. 认证接口

### 3.1 登录

代理商使用账号、密码获取 Access Token（及可选 Refresh Token、过期时间）。

**请求**

- **方法**：`POST`
- **路径**：`/auth/login`（或 `/api/agent/login`，以实际后端为准）
- **Body**：

```json
{
  "username": "agent_account",
  "password": "password_plain"
}
```

**响应**（成功时 `code === 0`）

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIs...",
    "refresh_token": "eyJhbGciOiJIUzI1NiIs...",
    "expires_in": 7200,
    "agent_id": "agent_001",
    "agent_name": "XX 区域代理商"
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| access_token | string | 访问令牌，后续接口放在 Authorization 头 |
| refresh_token | string | 刷新令牌，用于在 access_token 过期后换取新 Token，可选 |
| expires_in | int | access_token 有效时长（秒） |
| agent_id | string | 代理商 ID，可选，便于前端展示 |
| agent_name | string | 代理商名称，可选 |

**失败**：如账号或密码错误，`code !== 0`，例如 `code: 400`，`message: "账号或密码错误"`。

---

### 3.2 刷新 Token（可选）

当 access_token 即将过期或已过期时，使用 refresh_token 换取新的 access_token。

**请求**

- **方法**：`POST`
- **路径**：`/auth/refresh`（或 `/api/agent/refresh`)
- **Body**：

```json
{
  "refresh_token": "eyJhbGciOiJIUzI1NiIs..."
}
```

**响应**（成功）

```json
{
  "code": 0,
  "data": {
    "access_token": "new_eyJhbGciOiJIUzI1NiIs...",
    "expires_in": 7200
  }
}
```

若 refresh_token 无效或已过期，返回 401，exe 应清除本地 Token 并引导用户重新登录。

---

### 3.3 登出 / 失效 Token（可选）

客户端主动登出时调用，服务端将当前 Token 加入黑名单或删除会话，避免 Token 继续被使用。

**请求**

- **方法**：`POST`
- **路径**：`/auth/logout`
- **请求头**：`Authorization: Bearer <access_token>`

**响应**

```json
{
  "code": 0,
  "message": "success"
}
```

## 4. 授权相关接口

以下接口均需在请求头中携带有效 `access_token`，服务端根据 Token 识别代理商，并**仅返回或仅允许操作该代理商名下的数据**。

### 4.1 获取当前代理商名下设备列表

授权 exe 在「我的设备」中展示的列表，仅包含当前登录代理商名下的设备；生成 license 时，仅允许对列表中的设备签发。

**请求**

- **方法**：`GET`
- **路径**：`/api/agent/devices`（或 `/license/agent/devices`）
- **请求头**：`Authorization: Bearer <access_token>`
- **Query 参数**（可选）：分页与筛选

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | int | 否 | 页码，从 1 开始 |
| page_size | int | 否 | 每页条数，如 20 |
| serial | string | 否 | 按序列号模糊查询 |
| fingerprint | string | 否 | 按设备指纹精确匹配，用于导入 device_info 后校验是否在名下 |

**响应**（成功）

```json
{
  "code": 0,
  "data": {
    "list": [
      {
        "device_id": "dev_001",
        "serial": "RK3568-XXXX-001",
        "fingerprint": "a1b2c3d4e5f6...",
        "model": "RK3568",
        "bound_at": "2025-01-15T10:00:00Z",
        "remark": "某客户 KTV"
      }
    ],
    "total": 100
  }
}
```

| 字段 | 说明 |
|------|------|
| list | 当前代理商名下的设备数组 |
| device_id | 云端设备 ID，可选 |
| serial | 设备序列号，与 device_info.json 中 serial 对应 |
| fingerprint | 设备指纹，与 device_info.json 中 device.fingerprint 对应，用于生成 license 时的设备绑定与归属校验 |
| model / bound_at / remark | 展示用，可选 |

**exe 侧使用**：

- 导入 `device_info.json` 后，用其中的 `serial` 或 `device.fingerprint` 与 `list` 中项做匹配；匹配到则允许配置并生成 license.dat，否则提示「该设备不在您名下，无法授权」。

---

### 4.2 校验设备是否属于当前代理商（可选）

若不想先拉全量列表再本地匹配，可单独提供「按设备标识校验归属」的接口，供 exe 在导入 device_info 后调用。

**请求**

- **方法**：`GET` 或 `POST`
- **路径**：`/api/agent/device/check`（或 `/license/agent/device/check`）
- **请求头**：`Authorization: Bearer <access_token>`
- **Body（POST）或 Query（GET）**：

```json
{
  "serial": "RK3568-XXXX-001"
}
```

或按指纹校验：

```json
{
  "fingerprint": "a1b2c3d4e5f6..."
}
```

**响应**（成功且设备属于当前代理商）

```json
{
  "code": 0,
  "data": {
    "belongs": true,
    "device": {
      "device_id": "dev_001",
      "serial": "RK3568-XXXX-001",
      "fingerprint": "a1b2c3d4e5f6...",
      "model": "RK3568"
    }
  }
}
```

若设备不属于当前代理商或不存在：

```json
{
  "code": 0,
  "data": {
    "belongs": false,
    "device": null
  }
}
```

或直接返回 `code: 403`，`message: "该设备不在您名下"`，以实际后端约定为准。

---

### 4.3 上报授权签发记录（可选）

在 exe 成功生成 license.dat 后，可选调用本接口上报「某代理商对某设备在某时间签发了授权」，便于审计与对账。

**请求**

- **方法**：`POST`
- **路径**：`/api/agent/license/issue-record`（或 `/license/agent/issue-record`）
- **请求头**：`Authorization: Bearer <access_token>`
- **Body**：

```json
{
  "device_fingerprint": "a1b2c3d4e5f6...",
  "device_serial": "RK3568-XXXX-001",
  "customer_name": "XXX KTV",
  "expiry_time": 1710432000,
  "modules": ["ktv", "vod"]
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| device_fingerprint | string | 设备指纹，与 license.dat 中 device.fingerprint 一致 |
| device_serial | string | 设备序列号，可选 |
| customer_name | string | 客户名称，可选 |
| expiry_time | int | 授权到期时间（Unix 时间戳），可选 |
| modules | array | 授权模块列表，可选 |

**响应**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "record_id": "rec_001"
  }
}
```

上报失败（如网络问题）不应阻止 exe 本地已生成的 license.dat 使用，可仅做日志或重试队列。

## 5. 接口汇总表

| 接口 | 方法 | 路径（示例） | 认证 | 说明 |
|------|------|--------------|------|------|
| 登录 | POST | /auth/login | 否 | 账号密码获取 access_token |
| 刷新 Token | POST | /auth/refresh | 否 | refresh_token 换新 access_token |
| 登出 | POST | /auth/logout | 是 | 使当前 Token 失效 |
| 获取名下设备 | GET | /api/agent/devices | 是 | 当前代理商设备列表 |
| 校验设备归属 | GET/POST | /api/agent/device/check | 是 | 校验某设备是否属于当前代理商 |
| 签发记录上报 | POST | /api/agent/license/issue-record | 是 | 记录本次签发，用于审计 |

---

## 6. 授权 exe 与认证的配合

- **启动 / 恢复**：若本地无有效 Token 或 Token 已过期且刷新失败，则仅显示登录页，不允许进入「我的设备」或生成授权。
- **请求「名下设备」或「校验归属」**：均携带 `Authorization: Bearer <access_token>`；若返回 401，则清除本地 Token 并退回登录页。
- **生成 license.dat**：仅当设备通过「名下设备列表」或「校验设备归属」接口确认为当前代理商名下后，才允许配置并生成；生成后可选调用签发记录接口上报。

以上为云端认证与授权接口的设计说明，实际路径与字段名以云端后台实现为准，本文档可作为前后端对接与授权 exe 开发的依据。
