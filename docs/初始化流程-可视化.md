# 初始化流程（可视化文档）

## 设计原则：按图层驱动功能与硬件

启动顺序严格按以下链路，确保**只加载启用图层所需的功能模块与硬件端口**：

1. **授权文件分析** → 解析 license.dat，得到支持图层列表（enabled_layers）与模块列表（modules）
2. **构建授权图层池** → 使用 license.dat 的 enabled_layers 与内置 LayerDefinitions 生成可用图层定义
3. **创建授权图层** → 仅预创建授权图层池中的图层，默认隐藏
4. **根据 config.json 启用图层** → 对在 config 中有配置的图层应用参数（可见性、尺寸、采集等）
5. **根据启用图层加载功能模块与硬件端口** → DMX、采集、DSP、默认播放等按「授权 + config 启用图层」按需启动

## Step 2：启动引擎框架（Runtime）
├─ 2.1 使用已加载的 config（不再重复加载）
│  ├─ 布局、区域数、幕布分辨率、图层配置等已在 Step 1.3 加载到 SystemConfig
│  └─ 此处仅使用内存中的配置构造 Mubu 等
│
└─ 2.2 构造管理器骨架
   └─ Mubu（图层管理器）
   （LicenseManager 已在 Step 1.4 创建；其余管理器在 Step 3 中创建）

## Step 3：创建授权图层并依 config 启用
├─ 3.1 预创建授权图层
│  ├─ 从 LicenseManager 获取 enabled_layers，并由内置 LayerDefinitions 解析图层定义
│  ├─ 预创建这些图层，默认隐藏
│  └─ createLayersFromConfig() 对 config.json 中有的图层应用配置（得到「启用图层」）
│
├─ 3.2 应用图层配置（根据 config 启用）
│  ├─ 从 config.json 读取每个图层参数（尺寸、位置、可见性等）
│  ├─ 得到 config 启用图层列表 getConfiguredLayerIds()，供 Step 4 按需加载模块与端口
│  ├─ 视频图层：播放速率、音量、音轨
│  ├─ 图片图层：滤镜、淡入淡出、动画
│  └─ 文本图层：字体、颜色、对齐
│
├─ 3.3 创建其余管理器
│  ├─ PlaylistManager（播放列表）
│  ├─ SceneManager（场景管理）
│  ├─ CommandRouter（命令路由）
│  ├─ AudioProcessor（音频处理）
│  └─ EffectManager（特效管理）
│
└─ 3.4 准备图层资源（按需/内嵌）
   ├─ 播放器/解码器在图层首次播放时初始化
   └─ ASS 引擎（歌词图层）在图层使用中初始化

## Step 4：根据启用图层加载功能模块与端口
├─ 4.1 按授权与 config 启动硬件

│  └─ 否则不占用串口，日志输出「DMX512 未启用」
│
├─ 4.2 初始化渲染模块
│  ├─ VulkanRenderer 初始化（12步流程）
│  │  ├─ 创建 Vulkan 实例
│  │  ├─ 创建窗口表面
│  │  ├─ 选择物理设备（GPU）
│  │  ├─ 创建逻辑设备
│  │  ├─ 创建命令池
│  │  ├─ 创建交换链
│  │  ├─ 创建渲染通道
│  │  ├─ 创建描述符集布局
│  │  ├─ 创建图形管线
│  │  ├─ 创建帧缓冲
│  │  ├─ 创建命令缓冲
│  │  └─ 创建同步对象
│  ├─ Mubu 初始化（传入幕布分辨率）
│  ├─ 关联 Renderer 到所有图层
│  └─ RegionRotationRenderer 初始化（区域分割/旋转）
│
├─ 4.3 业务能力初始化
│  ├─ PlaylistManager 初始化（播放列表 DB）
│  ├─ VodDatabase 初始化（点播库，可选）
│  ├─ CommandRouter / SceneManager 依赖注入（Mubu、SystemConfig、RegionRotationRenderer 等）
│  └─ 区域配置从 SystemConfig 加载到 RegionRotationRenderer

├─ 4.4 DSP 音频配置

│  └─ 系统音量（从 config.json）
│
├─ 4.5 初始化网络模块（创建实例，不启动监听）
│  ├─ HttpServer(8080)      主 Web / 调试
│  ├─ HttpServer(8081)      移动端
│  ├─ HttpServer(8089)      点歌 /ktv、/vod
│  ├─ TcpServer(9000)       TCP 服务
│  ├─ UdpServer(8000)       UDP 服务

│
├─ 4.6 统一启用外部端口
│  ├─ HTTP :8080 主 Web
│  ├─ HTTP :8081 移动端
│  ├─ HTTP :8089 点歌 /ktv、/vod
│  ├─ TCP  :9000
│  ├─ UDP  :8000
│  └─ WebSocket :8090
│
├─ 4.6 后 渲染就绪
│  └─ initialized_ 置位（不早于 4.6，否则与 4.4/4.5/4.6 并行易导致黑屏更久）
│
├─ 4.7 默认播放

## Step 4.9：设备初始化上报（终端→授权服务）

在**网络端口已启用（4.6）之后**、不阻塞主流程的前提下，**异步**向授权服务上报本机设备信息与授权状态，用于后台设备列表、在线状态与启用/禁用管理。

├─ 4.9.1 触发时机
│  └─ 在 Step 4.6 完成（`initialized_` 置位）之后执行一次；可选：启动后延迟 2～5 秒再上报，避免与默认播放争用
│
├─ 4.9.2 配置
│  └─ `config.json` 中可选字段 `license_server_url`（如 `"http://60.205.127.117:8080"`）；为空或未配置则不发起上报
│      └─ 示例：`"license_server_url": "http://60.205.127.117:8080"`
│
├─ 4.9.3 收集数据
│  ├─ **设备信息**：型号、序列号、MAC、设备指纹（与 `/api/system/device-info` 一致）
│  │  └─ SystemUtils::getHardwareModel / getHardwareSerial / getMacAddress / getCpuSerial / getStorageSerial / generateDeviceFingerprint
│  └─ **授权信息**：授权开始时间、到期时间（Unix 秒，来自 LicenseManager；无授权时可传 0 或省略）
│
├─ 4.9.4 请求
│  ├─ 方法：`POST {license_server_url}/api/device/register`
│  ├─ Content-Type：`application/json`
│  └─ Body（与授权服务约定一致）：
│      {
│        "version": 1,
│        "export_time": <Unix 秒>,
│        "device": {
│          "model": "<型号>",
│          "serial": "<序列号>",
│          "cpu_serial": "<可选>",
│          "storage_serial": "<可选>",
│          "mac": "<MAC 地址>",
│          "fingerprint": "<设备指纹>",
│          "license_start": <可选，Unix 秒>,
│          "license_expiry": <可选，Unix 秒>
│        }
│      }
│
├─ 4.9.5 响应与行为
│  ├─ `code === 0`：上报成功；可选解析 `data.device_status`（enabled/disabled）用于本机提示
│  ├─ 网络失败或超时：仅打日志，不阻塞启动、不弹窗；可后续由「30 分钟心跳」重试
│  └─ 不依赖上报结果决定是否允许播放（播放仍由本地 license.dat 控制）
│
└─ 4.9.6 后续心跳（可选）
   └─ 终端每 30 分钟向同一接口再发一次相同结构的上报，用于刷新后台「在线」状态；实现方式可为定时器或后台线程

## 避免重复逻辑（已落实）

├─ **config.json**：仅在 Step 1.3 加载一次，Step 2.1 只使用内存中的 SystemConfig，不再次 load
├─ **授权图层池**：由 license.dat 的 enabled_layers 与内置 LayerDefinitions 构建，不读取 layers.json
├─ **图层 71 尺寸/位置**：仅在 Step 3.2 createLayersFromConfig 中应用；Step 4.2 只补加载二维码图片，不重复 setSize/setPosition
└─ **config 目录**：Step 1.2 已创建，Step 1.3 创建默认 config 时仅做防御性检查

## 注意事项

### 启动黑屏与默认播放
├─ 原因：此前 initialized_ 在 init 末尾才置位，整段 init 期间 update() 直接 return、不绘制
├─ 实现：在 4.6 完成后置位 initialized_（过早置位会与 4.4/4.5/4.6 并行，易致主线程争用、黑屏更久）；默认播放不再额外 sleep
├─ 效果：黑屏时间 ≈ Java 资源同步 + Native Step1～4.6 + 首帧解码
└─ 详见：**docs/启动黑屏-问题与优化.md**（链路说明、耗时日志 [Init 耗时]/[启动黑屏]、优化方向）

### 硬件与授权策略
├─ 关键硬件失败（GPU/存储）：终止启动，提示错误
├─ 可选硬件失败（串口）：继续启动，相关功能禁用
├─ 授权失败（license.dat 无效/缺失）：不终止启动；显示水印，15 天后限制视频播放
└─ 按图层加载：DMX、采集等仅在「授权模块 + config 启用图层」满足时启动，避免多余占用
