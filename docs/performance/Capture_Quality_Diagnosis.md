# 采集清晰度问题诊断报告

**设备**: 192.168.1.100  
**日期**: 2026-06-19  
**问题**: 采集画面清晰度不理想

---

## 🔍 关键发现

### 采集配置信息

从日志分析：

```
[采集][MIPI] Using request size 1920x1080 for pipeline/format
[采集][V4L2] Format Truth: 1920x1080, Stride 1920, Format NV12(0x3231564E)
[采集追踪][渲染入口] layer10 path=DMA rect=1402,73 420x945 fitMode=1
[采集追踪][Vulkan保持比例] tex=1920x1080 before=0.0,0.0 1920.0x1080.0 
                           after=720.0,0.0 480.0x1080.0
```

**关键信息**:
1. ✅ 采集分辨率: **1920x1080** (Full HD)
2. ✅ 采集格式: **NV12** (未压缩，YUV420)
3. ❌ 渲染目标: **420x945** (显示区域很小)
4. ❌ 缩放过程: **1920x1080 → 480x1080 → 420x945**

---

## 🎯 问题根源（已确认）

### 主要原因: 显示区域太小 ⭐⭐⭐

```
采集分辨率: 1920x1080 (输入)
   ↓
中间裁剪: 480x1080 (保持纵横比，裁剪左右)
   ↓  
最终显示: 420x945 (目标区域)
```

**问题**:
- 采集1920x1080的画面
- 但只显示在420x945的小区域
- **缩放比例**: 1920 → 420 = **缩小到22%**
- **信息损失**: 损失约78%的像素信息

**结论**: 
- ✅ 采集本身是清晰的（1080p）
- ❌ 显示时严重缩小，导致看起来不清晰

---

## 📊 详细分析

### 1. 采集参数（正常）✅

| 参数 | 值 | 状态 |
|------|-----|------|
| 输入分辨率 | 1920x1080 | ✅ 正常 |
| 采集格式 | NV12 (YUV420) | ✅ 未压缩 |
| 色彩空间 | Default | ✅ 正常 |
| 帧率 | Auto | ✅ 正常 |

### 2. 渲染参数（问题所在）❌

| 参数 | 值 | 状态 |
|------|-----|------|
| 显示区域 | 420x945 | ❌ 太小 |
| 缩放比例 | 22% | ❌ 严重缩小 |
| fitMode | 1 (填充裁剪) | ⚠️ 裁剪了内容 |

### 3. 缩放过程

```
原始: 1920x1080
  ↓
步骤1: 保持纵横比适配
  1920:1080 = 1.78
  目标: 420:945 = 0.44
  需要裁剪宽度
  ↓
步骤2: 裁剪到 480x1080 (左右各裁720px)
  ↓
步骤3: 缩放到 420x945
  480 → 420 (缩小12.5%)
  1080 → 945 (缩小12.5%)
```

**问题**: 
- 裁剪掉了左右各720像素（共1440像素）
- 然后缩小到目标区域
- **双重信息损失**

---

## 💡 解决方案

### 方案1: 增大显示区域 ⭐⭐⭐（最有效）

**问题**: 当前显示区域只有 420x945（约0.4平米）

**解决方案**: 增大图层显示区域

**在配置中修改**:
```json
{
  "layerId": 10,
  "coordinate": "1400 70 800 1800",  // 增大宽度和高度
  // 从 420x945 增大到 800x1800
}
```

**效果**: 
- 缩放比例: 1920 → 800 = 42% (vs 之前22%)
- 清晰度提升约 **2倍**

---

### 方案2: 修改fitMode避免裁剪 ⭐⭐

**问题**: 当前 `fitMode=1` (填充裁剪模式)

**解决方案**: 改为保持完整画面

```json
{
  "layerId": 10,
  "fitMode": 0,  // 改为保持比例，不裁剪
}
```

**效果**: 保留完整画面，不裁剪左右

---

### 方案3: 提升纹理过滤质量 ⭐

**检查CaptureRenderer的纹理过滤设置**

应该使用高质量过滤：
- ✅ `VK_FILTER_LINEAR` (线性过滤)
- ❌ `VK_FILTER_NEAREST` (最近邻，会有锯齿)

**修改位置**: `src/renderer/CaptureRenderer.cpp`

查找纹理采样器设置，确保使用线性或各向异性过滤。

---

### 方案4: 检查Shader质量 ⭐

**YUV到RGB转换的Shader质量**

确保使用高质量的色彩空间转换：

```glsl
// 应该使用完整的BT.709矩阵
mat3 yuv2rgb = mat3(
    1.164,  0.000,  1.596,
    1.164, -0.392, -0.813,
    1.164,  2.017,  0.000
);
```
