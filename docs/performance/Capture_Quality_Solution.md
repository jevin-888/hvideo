# 采集清晰度问题 - 完整解决方案

**设备**: 192.168.1.100  
**日期**: 2026-06-19  
**问题**: 采集画面清晰度不理想

---

## 🎯 问题总结

### 诊断结果

从设备日志分析：

```
采集分辨率: 1920x1080 (Full HD) ✅
采集格式: NV12 (未压缩YUV420) ✅
显示区域: 420x945 ❌ (太小)
缩放比例: 22% ❌ (严重缩小)
```

**关键发现**:
- ✅ 采集本身质量**很好**（1080p, 未压缩）
- ❌ 显示区域**太小**（只有420x945像素）
- ❌ 1920→420，缩小到**22%**，损失78%的信息

---

## 📊 缩放过程详解

```
原始采集: 1920x1080 (Full HD)
    ↓
保持比例裁剪: 480x1080 (裁掉左右各720px)
    ↓
缩放到目标: 420x945 (显示区域)
    ↓
结果: 只显示原画面的22%
```

**问题**:
1. fitMode=1 导致裁剪（丢失内容）
2. 显示区域太小（丢失细节）

---

## ✅ 解决方案

### 方案1: 增大显示区域 ⭐⭐⭐（最有效）

**当前配置**:
```
coordinate: "1402 73 420 945"
宽度: 420px
高度: 945px
```

**建议配置**:
```json
{
  "layerId": 10,
  "coordinate": "1000 70 1000 1800",
  "visible": true
}
```

**效果对比**:

| 参数 | 当前 | 优化后 | 改善 |
|------|------|--------|------|
| 显示宽度 | 420px | 1000px | +138% |
| 显示高度 | 945px | 1800px | +90% |
| 缩放比例 | 22% | 52% | +136% |
| **清晰度** | 差 | 好 | **+2.4倍** |

---

### 方案2: 修改fitMode避免裁剪 ⭐⭐

**当前设置**: `fitMode: 1` (填充裁剪模式)

```json
{
  "layerId": 10,
  "fitMode": 0  // 改为保持比例，不裁剪
}
```

**效果**:
- 保留完整的1920x1080画面
- 不裁剪左右内容
- 适应显示区域

---

### 方案3: 调整显示位置和大小的完整示例

**推荐配置**:
```json
{
  "layerId": 10,
  "name": "采集图层",
  "type": "VIDEO",
  "coordinate": "800 50 1200 2000",  // x, y, width, height
  "visible": true,
  "transparency": 255,
  "priority": 10,
  "fitMode": 0,  // 保持比例，不裁剪
  "videoPath": "/dev/video0",
  "captureType": "MIPI"
}
```

**效果**:
- 更大的显示区域（1200x2000）
- 保持完整画面
- 清晰度提升约**3倍**

## 📊 性能影响评估

### 增大显示区域的性能影响

| 显示大小 | 像素数 | 渲染耗时(估算) |
|---------|--------|---------------|
| 420x945 | 397K | ~2ms |
| 800x1800 | 1.44M | ~5ms | 
| 1200x2000 | 2.4M | ~8ms |

**结论**:
- 增大显示区域会增加渲染耗时
- 但影响不大（约增加3-6ms）
- 远小于之前优化节省的50ms
- **值得为清晰度做出的性能权衡**

## 💡 其他可能影响清晰度的因素

### 1. HDMI线材质量 ⭐
- 使用高质量HDMI线
- 线材不要过长（建议<3米）
- 检查接口是否松动

### 2. 信号源分辨率 ⭐⭐
- 确认信号源输出1080p
- 不要使用1080i（隔行扫描）
- 检查信号源的输出设置

### 3. 采集设备驱动 ⭐
- 确认驱动版本最新
- 检查内核参数配置
