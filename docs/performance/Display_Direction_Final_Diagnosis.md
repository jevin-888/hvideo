# 显示方向变化问题 - 最终诊断报告

**设备**: 192.168.1.100  
**监控时间**: 2026-06-19 14:10-14:50  
**问题**: 没有配置rotation，但显示方向有时会变化

---

## 🎯 问题确认

### 用户描述

- ❌ **没有配置任何rotation**
- ✅ **配置的是保持比例显示** (fitMode=0)
- ❌ **但显示方向有时会变**

### 实际运行状态

从监控日志发现：

```
实际运行配置:
  rect=2933,339 734x1556  (竖向显示区域)
  fitMode=1               ← 填充裁剪模式（不是0！）
  tex=1920x1080           (横向纹理)
  after=705.3,0.0 509.5x1080.0
```

**关键发现**: 
- ✅ rotation=0 (正确，没有旋转)
- ❌ **fitMode=1** (填充裁剪)，不是配置的0（保持比例）
- ❌ fitMode=1会裁剪内容以填充显示区域

---

## 💡 问题根源

### 核心问题：fitMode配置未生效

**您的配置**:
```json
{
  "layerId": 10,
  "fitMode": 0  // 保持比例，不裁剪
}
```

**实际运行**:
```
fitMode=1  // 填充裁剪模式
```

**结果**: fitMode=1导致的行为

---

## 📊 fitMode=1的行为分析

### 正常状态（信号正常）

```
输入纹理: 1920x1080 (横向)
显示区域: 734x1556 (竖向)
fitMode=1: 填充裁剪

计算过程:
  1. 要填充竖向区域734x1556
  2. 保持纹理高度1080不变
  3. 裁剪宽度：1920 → 509.5
  4. 左右各裁掉约705像素

结果:
  after=705.3,0.0 509.5x1080.0
  显示的是中间509.5像素宽的内容
```

### 信号丢失状态

```
输入纹理: 1x1 (占位纹理)
显示区域: 734x1556 (竖向)
fitMode=1: 填充裁剪

计算过程:
  1. 1x1纹理适配到734x1556
  2. 计算裁剪区域

结果:
  after=0.3,0.0 0.5x1.0  ← 完全不同！
  marked as "changed"
```

### 为什么看起来方向变了？

**原因**: fitMode=1裁剪的内容不同

```
手机竖屏内容在1920x1080横向纹理中:
┌────────────────────────────────┐
│     ┌────────────┐              │ 1920x1080
│     │            │              │ (横向纹理)
│     │  手机竖屏  │              │
│     │    内容    │              │
│     └────────────┘              │
└────────────────────────────────┘
        ↑
    中间部分

fitMode=1裁剪后:
- 正常时: 裁剪中间509.5像素
  → 可能正好包含手机内容
  
- 信号丢失/恢复后: 裁剪位置可能变化
  → 显示的内容不同
  → 看起来方向变了
```

## ✅ 解决方案

### 方案1: 确保fitMode=0生效 ⭐⭐⭐（推荐）

**问题**: 配置的fitMode=0没有生效

**检查**:
1. 配置文件中Layer 10是否设置了`"fitMode": 0`
2. 配置是否正确读取
3. 是否有其他地方覆盖了fitMode

**修改配置**:
```json
{
  "layerId": 10,
  "coordinate": "2933 339 734 1556",
  "fitMode": 0,     // ← 确保是0
  "visible": true,
  "rotation": 0     // ← 明确设置为0
}
```

**验证**:
```bash
# 重启应用后检查
adb logcat | grep "layer10.*fitMode"

# 应该看到：
# fitMode=0 (不是1)
```

---

### 方案2: 如果fitMode必须是1，则设置rotation=90 ⭐⭐

**如果**需要使用fitMode=1（填充裁剪），**则**需要配置rotation：

```json
{
  "layerId": 10,
  "coordinate": "2933 339 734 1556",
  "fitMode": 1,       // 填充裁剪
  "rotation": 90,     // ← 旋转纹理
  "visible": true
}
```

**效果**: 
- 先旋转纹理90度
- 然后再裁剪适配
- 显示方向正确

---

### 方案3: 调整显示区域为横向 ⭐

**如果不旋转**，可以改为横向显示区域：

```json
{
  "layerId": 10,
  "coordinate": "x y 1556 734",  // 横向区域
  "fitMode": 0,
  "visible": true
}
```

## 🔍 为什么fitMode=1没有按预期工作？

### fitMode=0 vs fitMode=1

**fitMode=0 (保持比例)**:
```
输入: 1920x1080
显示: 734x1556
结果: 
  - 保持宽高比16:9
  - 适配到734x1556
  - 上下或左右留黑边
  - 不裁剪内容 ✅
```

**fitMode=1 (填充裁剪)**:
```
输入: 1920x1080  
显示: 734x1556
结果:
  - 填充整个显示区域
  - 裁剪多余部分
  - 无黑边
  - 会丢失内容 ❌
```

### 手机投屏的正确配置

```
手机竖屏内容 (1080x1920) 
  ↓ HDMI输出
横向采集 (1920x1080)
  ↓ 显示
需要旋转或正确适配

推荐配置:
1. fitMode=0 + rotation=90  ← 最佳
2. fitMode=0 + 横向显示区域
3. 不推荐fitMode=1（会裁剪）
```
