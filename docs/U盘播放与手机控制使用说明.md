# U 盘播放与手机控制使用说明

本文档说明如何在 Hvideo 设备上使用 **U 盘作为媒体源**进行播放，并通过 **手机** 进行发现设备、控制播放与点歌。

---

## 一、U 盘播放

### 1.1 使用方式概览

| 方式 | 说明 | 适用场景 |
|------|------|----------|
| **方式 A** | 将 U 盘内视频/图片拷贝到设备内置存储的 `huoshan` 目录，按现有播放列表使用 | 内容固定、追求稳定 |
| **方式 B** | 在播放列表中直接使用 U 盘挂载路径作为媒体 `uri`（若系统支持该路径访问） | 即插即播、不占内置空间 |

### 1.2 U 盘在设备上的挂载路径（Android/嵌入式）

- 常见挂载点（因系统/厂商而异，以实际为准）：
  - `/storage/XXXX-XXXX/`（如 UUID 卷名）
  - `/mnt/usb/`、`/mnt/usb_storage/`
  - `/storage/usb/`、`/sdcard/usb/`
- 插入 U 盘后，可在设备终端执行 `df` 或 `mount` 查看实际挂载点。

### 1.3 方式 A：拷贝到 huoshan 目录播放（推荐）

1. **设备根目录**  
   应用数据根目录 `ROOT_PATH` 由启动路径配置决定：优先已有 `/huoshan/`，找不到时才使用 `/sdcard/huoshan/`。  
   详见 [根目录语义路径](./根目录语义路径.md)。

2. **目录约定**  
   - 视频：放到 `ROOT_PATH/video/` 下，例如：`ROOT_PATH/video/宣传片.mp4`。  
   - 图片：放到 `ROOT_PATH/Image/` 下。  
   - 歌词：`ROOT_PATH/Lyrics/`。  

3. **播放列表中的 uri**  
   使用**相对语义路径**即可，例如：
   - 视频：`/video/宣传片.mp4` 或 `video/宣传片.mp4`（由系统按 ROOT_PATH 解析）。

4. **操作步骤简述**  
   - 将 U 盘插到电脑，把需要的视频/图片拷贝到 U 盘。  
   - 把 U 盘插到 Hvideo 设备，再把 U 盘里的文件拷贝到设备的 `ROOT_PATH/video/`（或 Image、Lyrics 等）。  
   - 通过手机控制端创建/编辑播放列表，添加上述路径的媒体并播放。

### 1.4 方式 B：播放列表中直接使用 U 盘路径

- 播放列表项使用 `uri` 字段，可为**绝对路径**（若应用有权限访问）。  
- 示例：U 盘挂载在 `/storage/ABCD-1234/`，则可在添加播放列表项时使用：
  - `"uri": "/storage/ABCD-1234/视频/片头.mp4"`
- **注意**：  
  - 需确认应用对 U 盘挂载目录有读权限（Android 存储权限/MANAGE_EXTERNAL_STORAGE 等）。  
  - U 盘拔出后，对应 uri 会失效，需重新插盘或改用方式 A。

### 1.5 播放列表与 U 盘配合的典型流程

1. 用手机或 Web 控制端调用 **创建播放列表** 接口，得到 `playlist_id`。  
2. 调用 **添加项** 接口，为每个 U 盘内（或已拷贝到 huoshan 的）视频添加一项，`uri` 为语义路径或 U 盘绝对路径。  
3. 调用 **播放播放列表** 或 **播放指定项** 接口，指定 `layer_id`（如主视频图层 1）。  
4. 后续用手机进行暂停/继续/重播/下一曲等控制即可。

## 二、手机控制

### 2.1 发现设备（同一局域网）

- 设备通过 **UDP 广播** 在 LAN 内被发现，手机与设备需在同一网段。  
- 端口：**18080 (UDP)**。  
- 协议说明见 [网络设备发现协议](./网络设备发现协议.md)。

**手机端两种用法：**

1. **被动发现**  
   手机监听 18080 端口，设备每约 3 秒发送一次 Beacon，手机解析 JSON 得到设备 IP 和端口。

2. **主动查询**  
   手机向 `255.255.255.255:18080` 发送 JSON `{"type":"discover","query":"hvideo"}`，设备回复单播包。

**Beacon/响应示例：**

```json
{
  "type": "hvideo",
  "protocol": "discover",
  "version": "1.0",
  "ip": "192.168.1.100",
  "device_name": "hostname",
  "ports": {
    "http": 8080,
    "mobile": 8081,
    "vod": 8089,
    "ws": 8090,
    "tcp": 9000,
    "udp": 8000
  }
}
```

| 端口用途 | 端口号 | 说明 |
|----------|--------|------|
| 主 Web 播控 | 8080 | HTTP API、Web 控制台 |
| 移动端 | 8081 | 移动端专用入口 |
| 点歌 / KTV | 8089 | 点歌 Web UI，重定向到 /vod/ |
| WebSocket | 8090 | 实时控制、状态推送 |
| TCP 命令 | 9000 | 二进制命令 |
| UDP 命令 | 8000 | UDP 命令 |

发现到设备后，记下 `ip` 和 `ports`，后续用 `http://{ip}:8080`、`http://{ip}:8089` 等访问。

### 2.2 通过浏览器用手机控制（推荐）

1. 手机连上与设备同一 WiFi。  
2. 若支持发现：打开支持 UDP 发现的自定义页面/App，获取设备 IP；否则在路由器或设备上查看 IP，手动输入。  
3. 在手机浏览器访问：  
   - **播控/配置**：`http://{设备IP}:8080/`  
   - **点歌**：`http://{设备IP}:8089/` 或 `http://{设备IP}:8089/vod/`  
4. 在 Web 界面中进行播放、播放列表、点歌、切歌等操作。

### 2.3 常用 HTTP API（手机 App 或脚本调用）

基地址：`http://{设备IP}:8080`。

| 功能 | 方法 | 路径/说明 |
|------|------|-----------|
| 发现信息 | GET | `/api/system/discovery` |
| 心跳 | GET | `/api/heartbeat` |
| 播放列表列表 | GET | `/api/playlists` |
| 播放列表项 | GET | `/api/playlist/{id}/items?layer_id=1` |
| 创建播放列表 | POST | `/api/playlist/create`（body 含 `id`、`items` 等） |
| 添加项 | POST | `/api/playlist/{id}/item`（body 含 `uri`、`title` 等） |
| 播放播放列表 | POST | `/api/playlist/play`（body 含 `playlist_id`、`layer_id` 等） |
| 播放控制 | POST | 通过统一命令接口或 REST：播放/暂停/停止/下一首等 |

播放列表项中的 `uri` 可为：

- 相对语义路径：如 `video/宣传片.mp4`（对应 ROOT_PATH 下的 `video/` 目录）；  
- 或 U 盘绝对路径：如 `/storage/XXXX-1234/视频/片头.mp4`（需设备支持并有权访问）。

更多接口见 [开发文档](./开发文档.md) 与 [05-API参考文档](./05-API参考文档.md)。

### 2.4 TCP/WebSocket 命令（可选）

- **TCP**：连接 `{设备IP}:9000`，发送二进制或 JSON 命令（类型码 0x09 为播放列表，0x02 为视频播放控制等）。  
- **WebSocket**：连接 `ws://{设备IP}:8090`，用于实时控制与状态订阅。  

协议细节见 [开发文档](./开发文档.md) 中的命令码与参数说明。

## 三、典型使用流程小结

1. **准备 U 盘**  
   在电脑上把视频/图片拷入 U 盘（建议建 `video`、`Image` 等子目录便于管理）。

2. **接入设备**  
   - 将 U 盘插入 Hvideo 设备；  
   - 可选：把 U 盘内容拷贝到设备 `ROOT_PATH/video/` 等目录（方式 A）。

3. **手机发现设备**  
   同一 WiFi 下，通过发现协议或手动输入 IP，得到设备地址（如 `192.168.1.100`）。

4. **创建并播放列表**  
   - 浏览器打开 `http://192.168.1.100:8089/` 点歌，或 `http://192.168.1.100:8080/` 做播控；  
   - 或调用 API：创建播放列表 → 添加项（uri 为 U 盘路径或 `video/xxx.mp4`）→ 播放。

5. **日常控制**  
   用手机浏览器或集成 API 的 App 进行暂停、继续、切歌、点歌、调节音量等。
