# HSVJ 优化脚本安全说明

## ⚠️ 已修复的危险操作

原优化脚本 `hsvj_optimize.sh` 包含了大量 **卸载系统应用** 的操作，这些操作可能导致 Android 系统无法启动。

### 🔴 极度危险（会导致系统崩溃）

以下应用**绝对不能卸载**，已从脚本中移除：

#### 1. 系统 Provider（数据提供者）
```bash
# ❌ 危险 - 已移除
pm uninstall --user 0 com.android.providers.telephony      # 电话/短信数据
pm uninstall --user 0 com.android.providers.contacts       # 联系人数据
pm uninstall --user 0 com.android.providers.calendar       # 日历数据
pm uninstall --user 0 com.android.providers.downloads.ui   # 下载管理
pm uninstall --user 0 com.android.providers.partnerbookmarks
```
**后果**：系统框架崩溃，无法启动

#### 2. WebView（网页渲染引擎）
```bash
# ❌ 危险 - 已移除
pm uninstall --user 0 com.google.android.webview
```
**后果**：所有使用 WebView 的应用崩溃（包括很多系统应用）

#### 3. 网络核心组件
```bash
# ❌ 危险 - 已移除
pm uninstall --user 0 com.android.pacprocessor    # 代理自动配置
pm uninstall --user 0 com.android.proxyhandler    # 代理处理
pm uninstall --user 0 com.android.vpndialogs      # VPN 对话框
```
**后果**：网络功能异常，可能导致系统服务崩溃

#### 4. 系统 UI 核心组件
```bash
# ❌ 危险 - 已移除
pm uninstall --user 0 com.android.internal.systemui.navbar.gestural
pm uninstall --user 0 com.android.internal.systemui.navbar.threebutton
```
**后果**：导航栏消失，无法操作系统

#### 5. 文档管理
```bash
# ❌ 危险 - 已移除
pm uninstall --user 0 com.android.documentsui
```
**后果**：文件选择器崩溃，很多应用无法打开文件

#### 6. 证书和安全
```bash
# ❌ 危险 - 已移除
pm uninstall --user 0 com.android.certinstaller
```
**后果**：HTTPS 证书问题，网络应用无法正常工作

#### 7. 输入法
```bash
# ❌ 危险 - 已移除
pm uninstall --user 0 com.android.inputmethod.latin
```
**后果**：如果是唯一输入法，将无法输入文字

#### 8. 备份和存储
```bash
# ❌ 危险 - 已移除
pm uninstall --user 0 com.android.sharedstoragebackup
pm uninstall --user 0 com.android.localtransport
```
**后果**：系统备份功能异常

## ✅ 新版本安全策略

### 改用 `pm disable-user` 替代 `pm uninstall`

**区别：**
- `pm uninstall --user 0`：**永久删除**应用，无法恢复（除非重刷系统）
- `pm disable-user --user 0`：**禁用**应用，可以随时恢复

**恢复方法：**
```bash
# 恢复被禁用的应用
pm enable com.android.music
```

### 安全禁用的应用类别

新版脚本只禁用以下**确认安全**的应用：

#### ✅ 日常应用（不影响系统运行）
- 音乐播放器 (com.android.music)
- 图库 (com.android.gallery3d)
- 计算器 (com.android.calculator2)
- 时钟 (com.android.deskclock)
- 日历 (com.android.calendar)
- 录音机 (com.android.soundrecorder)

#### ✅ 相机
- com.android.camera2
- com.android.rockchip.camera2

#### ✅ 壁纸和屏保
- 壁纸选择器
- 动态壁纸
- 屏保

#### ✅ 打印服务
- com.android.printspooler
- com.android.bips

#### ✅ 主题包（13 个颜色主题 + 6 个图标形状 + 30 个图标包）
- 不影响系统运行，只是视觉效果

#### ✅ 测试工具
- 压力测试
- 设备测试
- 自动关机/启动工具

## 🛠️ 如何恢复系统

### 如果设备已经无法启动

#### 方案 1：Recovery 模式恢复出厂设置
```bash
# 1. 断开其他设备
adb disconnect 172.23.173.152:5555

# 2. 进入 recovery shell
adb -s 5ffe773386f7102a shell

# 3. 恢复出厂设置（会清除数据）
wipe data
wipe cache
reboot system
```

#### 方案 2：尝试重新安装被卸载的应用
如果系统还能进入 recovery shell：
```bash
adb -s 5ffe773386f7102a shell

# 尝试重新安装系统应用（需要有 APK 文件）
pm install -r /system/app/WebView/WebView.apk
```

#### 方案 3：重刷系统固件
如果上述方法都失败，需要重新刷入完整的 ROM。

## 📋 使用建议

### 1. 首次使用前备份
```bash
# 备份当前系统
adb backup -all -apk -shared -system
```

### 2. 分阶段测试
不要一次性执行所有优化，建议分步测试：
```bash
# 第一步：只执行基础优化
base_optimize

# 第二步：禁用少量应用测试
pm disable-user --user 0 com.android.music

# 第三步：确认无问题后再批量禁用
disable_bloatware
```

### 3. 保留恢复脚本
创建恢复脚本 `hsvj_restore.sh`：
```bash
#!/system/bin/sh
# 恢复所有被禁用的应用

pm enable com.android.music
pm enable com.android.gallery3d
pm enable com.android.calculator2
# ... 其他应用
```

## ⚡ 性能优化建议

除了禁用应用，还可以通过以下方式优化性能：

### 1. CPU 调度优化（安全）
```bash
# 性能模式
echo performance > /sys/devices/system/cpu/cpufreq/policy0/scaling_governor
```

### 2. 内存管理（安全）
```bash
# 清理缓存
echo 3 > /proc/sys/vm/drop_caches
```

### 3. OOM 保护（安全）
```bash
# 保护关键进程
echo -1000 > /proc/$(pidof com.hsvj.engine)/oom_score_adj
```

### 4. 日志降噪（安全）
```bash
setprop log.tag.V4L2DeviceDetector WARN
setprop log.tag.CompositionEngine WARN
```

## 📞 故障排查

### 问题：设备卡在 Recovery 模式
**原因**：系统关键组件被卸载  
**解决**：恢复出厂设置

### 问题：ADB 提示 "more than one device"
**原因**：多个设备连接  
**解决**：
```bash
adb disconnect 172.23.173.152:5555
adb -s 5ffe773386f7102a shell
```

### 问题：应用崩溃
**原因**：依赖的系统组件被禁用  
**解决**：
```bash
# 恢复 WebView
pm enable com.google.android.webview
```
