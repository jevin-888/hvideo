# 二维码图层功能文档

## 概述

二维码图层（Layer71）支持在前端生成二维码图片，保存到软件目录的二维码目录，然后通过Vulkan纹理渲染显示在图层上。

## 功能说明

二维码图层提供以下功能：

1. **前端生成**：使用 QRCode.js 库在前端实时生成二维码图片
2. **实时预览**：在生成前可以预览二维码效果
3. **参数配置**：支持内容、尺寸、纠错级别、前景色、背景色等参数
4. **自动保存**：生成的二维码自动保存到指定目录
5. **自动加载**：保存后自动加载到图层显示

---

## 实现流程

### 1. 前端生成阶段

```
用户输入内容 → QRCode.js生成Base64图片 → 发送到后端
```

- 前端使用 QRCode.js 库生成二维码
- 支持自定义尺寸、颜色、纠错级别
- 生成后转换为 Base64 格式发送给后端

### 2. 后端保存阶段

```
接收Base64数据 → 解码为二进制 → 保存为PNG文件 → 加载到图层
```

- 后端接收前端发送的 Base64 图片数据
- 解码为二进制数据
- 保存到 `QRCode/` 目录，文件名格式：`qrcode_{layerId}.png`
- 自动加载图片到图层

### 3. 渲染显示阶段

```
PNG文件 → stb_image解码 → Vulkan纹理 → GPU渲染
```

- 使用 `LayerImage` 类加载 PNG 图片
- 通过 `stb_image` 库解码为 RGBA 数据
- 创建 Vulkan 纹理
- 通过 Vulkan 图形管线渲染到屏幕

## 参数配置

### 二维码参数区域布局

二维码参数区域分为三个部分：

1. **左侧区域**：基础参数配置
2. **中间区域**：二维码生成设置
3. **右侧区域**：二维码预览

### 左侧区域 - 基础参数

| 参数名 | 类型 | 默认值 | 说明 | 可选值/范围 |
|--------|------|--------|------|------------|
| `qr_text` | string | `"微信扫码关注"` | 底部显示文字 | UTF-8字符串 |
| `qr_logo_path` | string | `""` | Logo图片路径 | 图片文件路径 |
| `qr_bg_color` | string | `"0.545 0.643 0.851 1.0"` | 背景颜色 | RGBA格式，范围 0.0-1.0 |
| `qr_content` | string | `""` | 地址内容（二维码内容） | UTF-8字符串 |
| `priority` | int | `71` | 优先级 | `0-100` |
| `position.x` | int | `0` | X坐标 | `≥0` |
| `position.y` | int | `0` | Y坐标 | `≥0` |
| `size.width` | int | `1920` | 宽度 | `≥0` |
| `size.height` | int | `1080` | 高度 | `≥0` |

### 中间区域 - 二维码生成设置

| 参数名 | 类型 | 默认值 | 说明 | 可选值/范围 |
|--------|------|--------|------|------------|
| `qr_size` | int | `256` | 二维码尺寸（像素） | `64-2048` |
| `qr_logo_size` | int | `0` | Logo尺寸（百分比） | `0-30` |
| `qr_error_correction` | int | `1` | 容错级别 | `0`: L (低/约7%)<br>`1`: M (中/约15%)<br>`2`: Q (高/约25%)<br>`3`: H (最高/约30%) |

### 其他参数

| 参数名 | 类型 | 默认值 | 说明 | 可选值/范围 |
|--------|------|--------|------|------------|
| `qr_fg_color` | string | `"0.0 0.0 0.0 1.0"` | 前景颜色（二维码图案颜色） | RGBA格式，范围 0.0-1.0 |
| `qr_text_color` | string | `"1.0 1.0 1.0 1.0"` | 底部文字颜色 | RGBA格式，范围 0.0-1.0 |

### 配置示例

```json
{
  "layer71": {
    "visible": true,
    "position": "0 0",
    "size": "1920 1080",
    "rotation": 0.0,
    "scale": 1.0,
    "alpha": 1.0,
    "priority": 71,
    "qr_content": "微信扫码关注",
    "qr_size": 256,
    "qr_logo_size": 0,
    "qr_error_correction": 1,
    "qr_fg_color": "0.0 0.0 0.0 1.0",
    "qr_bg_color": "0.545 0.643 0.851 1.0",
    "qr_text": "微信扫码关注",
    "qr_text_color": "1.0 1.0 1.0 1.0",
    "qr_logo_path": ""
  }
}
```

### 参数区域说明

**左侧区域（基础参数）：**
- **底部文字**：输入框，默认值"微信扫码关注"
- **Logo图**：上传logo按钮（紫色按钮）
- **边框颜色**：颜色选择器 + 十六进制输入框（默认值：#0051FF，对应RGBA：0 0.2 1 1.0）
- **地址内容**：输入框，二维码编码的内容（默认值"微信扫码关注"）
- **优先级**：数值输入框，带"-"和"+"按钮（默认值：71）
- **X**：数值输入框，带"-"和"+"按钮（默认值：0）
- **Y**：数值输入框，带"-"和"+"按钮（默认值：0）
- **宽度**：数值输入框，带"-"和"+"按钮（默认值：1920）
- **高度**：数值输入框，带"-"和"+"按钮（默认值：1080）

**中间区域（二维码生成设置）：**
- **尺寸**：数值输入框，带"-"和"+"按钮（默认值：256，范围：64-2048）
- **Logo尺寸(%)**：数值输入框，带"-"和"+"按钮（默认值：0，范围：0-30）
- **容错级别**：下拉选择框，选项：L (低)、M (中)、Q (高)、H (最高)，默认值：M (中)
- **生成二维码**：紫色按钮，用于生成二维码

**右侧区域（预览）：**
- 二维码预览卡片，显示生成的二维码和底部文字

## 路径配置

### 二维码存储目录

二维码文件保存在当前数据根目录下：

- **当前目录**：`ROOT_PATH/QRCode/`
- **根目录选择**：优先已有 `/huoshan/`，找不到 `/huoshan/` 时才使用 `/sdcard/huoshan/`

### 路径兼容性

系统会自动检测并选择当前 `ROOT_PATH`：

1. 如果设备已存在 `/huoshan/`，直接使用 `/huoshan/`
2. 只有找不到 `/huoshan/` 时，才创建并使用 `/sdcard/huoshan/`
3. 保存和读取都通过 `QR_CODE_DIR`，不在业务代码里手写具体根目录

### 文件命名规则

二维码文件命名格式：`qrcode_{layerId}.png`

示例：
- Layer71 的二维码文件：`qrcode_71.png`
- Layer60 的二维码文件：`qrcode_60.png`

### 目录结构示例

```
/huoshan/
├── QRCode/
│   ├── qrcode_71.png
│   └── qrcode_60.png
├── Image/
├── Layout/
└── ...
```

## 渲染方式

### 渲染流程

二维码使用 **Vulkan 纹理渲染**方式显示：

1. **图片加载**：使用 `LayerImage::loadImage()` 加载 PNG 文件
2. **图像解码**：通过 `stb_image` 库将 PNG 解码为 RGBA 数据
3. **纹理创建**：使用 `VulkanRenderer::createTextureFromRGBA()` 创建 Vulkan 纹理
4. **纹理渲染**：通过 `VulkanRenderer::renderLayer()` 渲染到屏幕

### 渲染特性

- **高性能**：直接使用 GPU 纹理渲染，性能优异
- **支持变换**：支持位置、尺寸、旋转、缩放、透明度等变换
- **实时更新**：二维码更新后自动重新加载和渲染

### 代码实现位置

| 功能 | 文件 | 说明 |
|------|------|------|
| 前端生成 | `src/network/web/modules/pages/layerMatrix.js` | QRCode.js 生成二维码 |
| 后端保存 | `src/core/CommandRouter.cpp` | 处理 `generate_qrcode` 命令 |
| 图片加载 | `src/layer/LayerImage.cpp` | 加载 PNG 文件 |
| 纹理创建 | `src/renderer/VulkanRenderer.cpp` | 创建 Vulkan 纹理 |
| 路径配置 | `src/core/PathConfig.cpp` | 路径初始化和管理 |

## 注意事项

### 1. 路径一致性

- 保存和读取统一使用当前 `ROOT_PATH/QRCode/`
- 代码中统一使用 `QR_CODE_DIR` 常量，不再自行判断 `/huoshan/` 与 `/sdcard/huoshan/`

### 2. 文件大小限制

- 二维码尺寸范围：64-2048 像素
- 建议尺寸：256-512 像素，平衡清晰度和文件大小

### 3. 纠错级别选择

- **L (0)**：约7%错误可纠正，适合简单场景
- **M (1)**：约15%错误可纠正，**推荐默认值**
- **Q (2)**：约25%错误可纠正，适合复杂场景
- **H (3)**：约30%错误可纠正，适合需要高容错率的场景

### 4. 颜色配置

- 前景色和背景色需要有足够的对比度，确保二维码可扫描
- 颜色格式：`"R G B A"`，每个值范围 0.0-1.0
- 示例：黑色前景 `"0.0 0.0 0.0 1.0"`，白色背景 `"1.0 1.0 1.0 1.0"`

### 5. 图层属性

二维码图层（Layer71）继承图像图层的所有通用属性：
- `visible`：可见性
- `position`：位置坐标
- `size`：图层尺寸
- `rotation`：旋转角度
- `scale`：缩放比例
- `alpha`：透明度

### 6. 自动加载

- 二维码生成后会自动加载到图层
- 如果图层已存在二维码文件，启动时会自动加载
- 文件路径保存在图层配置的 `image_path` 字段中
