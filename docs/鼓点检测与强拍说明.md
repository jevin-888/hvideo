# 鼓点检测与强拍说明

## 节拍检测：仅 RKNN

- **鼓点来源**：只使用 `WaveBeatDetector` 的 RKNN 模式；模型加载成功才有节拍检测。
- **未加载时**：`process()` 直接返回无节拍，闪黑/闪白不会随鼓点触发。
- **强拍**：直接使用 RKNN 输出的 `isDownbeat`（下拍）。

---

## 相关模块关系

| 模块 | 作用 |
|------|------|
| **WaveBeatDetector (RKNN)** | 唯一节拍源：输出 `isBeat`、`isDownbeat`、BPM。无模型则不输出 beat。 |
| **EffectManager::BeatTrackState** | 仍会收到 beat 并更新 BPM/小节；闪黑触发只看 `isDownbeat`。 |
| **DenseDetector** | 仅判断是否进入「密集鼓点」模式（快速闪白），不参与单次鼓点时刻。 |

---

## 使用前提

- 编译时开启 `ENABLE_RKNN`（见下方「出现 RKNN未启用 时」）。
- 部署并成功加载 `models/wavebeat_rk3588.rknn`。
- 日志中看到「RKNN模型初始化成功」即表示节拍检测已启用。

---

## 出现「RKNN未启用 / 未加载」时

日志示例：

```text
[WaveBeat] RKNN未启用，节拍检测仅支持 RKNN 模式
[AudioEffect] WaveBeat检测器初始化: RKNN=未加载(节拍检测不可用)
```

**原因**：编译时未带 RKNN SDK（CMake 未收到 `RKNN_SDK_PATH`），因此节拍检测不可用。**之前若好用**，多半是 gradle.properties 或环境变量里配过路径，换机器/重拉代码后需再配一次。
1. **准备 RKNN 运行时 SDK**（瑞芯微 NPU，如 RK3588 用 rknn-toolkit2 对应 runtime）  
   - 需要包含：`include/` 头文件、`lib/librknnrt.so`（Android 用 arm64-v8a）。

2. **任选一种方式传入 SDK 路径**（之前若好用，多半是下面某一种配过，换环境/重拉代码后补上即可恢复）  
   - **方式 A（推荐）**：在工程根目录 **gradle.properties** 里增加一行（路径改成你本机实际路径）：
   ```properties
   rknnSdkPath=C\:\\你的路径\\rknpu2\\runtime\\Android\\arm64-v8a
   ```
   相对路径也可，例如：`rknnSdkPath=Third-Party/rknn-android/arm64-v8a`  
   - **方式 B**：设置环境变量 **RKNN_SDK_PATH** 再编译（CMake 会读该变量）。  
   - **方式 C**：把 SDK 放到 `Third-Party/rknpu2/runtime/Android/arm64-v8a` 等常见路径，CMake 会自动检测（见根目录 CMakeLists.txt 中的 _rknn_candidates）。

3. **重新编译并安装 APK**  
   - 配置好后 **Clean + Rebuild**。编译成功启用时会看到：`RKNN SDK enabled for WaveBeat: ...`。

4. **确认模型文件**  
   - 应用从 assets 读 `models/wavebeat_rk3588.rknn`；若缺失会打「RKNN模型加载失败」而不是「RKNN未启用」。

总结：**「RKNN未启用」= 编译时未带 RKNN SDK。** 之前若好用，多半是 gradle.properties 或环境变量里配过路径，换环境或重拉代码后需再配一次。按上面任选一种方式配好并 Clean 重新编译即可恢复。

## RK3566/RK3568 上播放时崩溃（SIGSEGV in WaveBeatDetector::process / resampleAndBuffer）

部分 RK3566/RK3568 设备会在**重采样循环**或 **rknn_run** 内崩溃。若日志出现「RKNN=已加载」但一播放就闪退，可**临时关闭节拍**以保稳定：

- **关闭整个节拍检测（推荐，旧 RKNN 版本）**：在设备当前 `ROOT_PATH` 下创建 **`.disable_wavebeat_rknn`**
  - 例：`adb shell 'ROOT=$(if [ -d /huoshan ]; then echo /huoshan; else echo /sdcard/huoshan; fi); touch "$ROOT/.disable_wavebeat_rknn"'`
  - 重启应用后日志会提示「已关闭节拍检测（跳过重采样与推理）」；不再进入 resample/推理，应用可稳定运行。
- **仅跳过重采样（仍做推理，旧 RKNN 版本）**：在设备当前 `ROOT_PATH` 下创建 **`.disable_wavebeat_resample`**
  - 例：`adb shell 'ROOT=$(if [ -d /huoshan ]; then echo /huoshan; else echo /sdcard/huoshan; fi); touch "$ROOT/.disable_wavebeat_resample"'`
  - 不再执行重采样，仅用已有缓冲做 RKNN 推理（效果会偏差，但可验证是否仅为 resample 崩溃）。
- 删除对应文件并重启即可恢复。

## 逻辑小结

| 检查项 | 状态 |
|--------|------|
| 节拍来源 | 唯一：`WaveBeatDetector::process()` → RKNN 推理，无 Fallback |
| 强拍 | 仅用 `beatResult.isDownbeat`（RKNN 输出） |
| 闪黑 | 仅强拍触发；多图层共享同一 peak 标志 |
