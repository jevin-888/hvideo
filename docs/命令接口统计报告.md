# 命令接口统计报告

## 概述

本报告统计了项目中所有命令接口的数量和详细清单。项目采用命令路由模式，通过命令码（code）和动作（action）来组织接口。

**统计时间**: 2026年6月3日

## 总体统计

- **命令码总数**: 13个（0x00、0x01、0x02、0x03、0x06、0x08、0x09、0x0A、0x0C、0x0D、0x10、0x50、0x51）
- **接口总数**: 约100+个（按 action 统计）
- **HTTP端点**: 1个通用命令接口 `/api/command`

## 命令接口清单

### 0x00 - 系统配置（SystemConfig）

**处理函数**: `handleSystemConfig`

**接口列表**:
1. `status` - 查询系统状态（分辨率、音频类型、设备类型等）
2. 直接参数设置（无action）：
   - `resolution` - 设置分辨率
   - `audio_type` - 设置音频类型
   - `device_type` - 设置设备类型
   - `screen_rotate` - 设置屏幕旋转

**接口数量**: 1个查询接口 + 4个配置参数 = 5个功能点

---

## HTTP接口

### 通用命令接口

**路径**: `POST /api/command`

**请求格式**:
```json
{
  "type": 0,
  "code": 0x01,
  "request_id": "unique-request-id",
  "timestamp": 1234567890,
  "param": {
    "action": "list",
    "layer_id": 1,
    ...
  }
}
```

**响应格式**:
```json
{
  "code": 0x01,
  "request_id": "unique-request-id",
  "timestamp": 1234567890,
  "trace_id": "trace-id",
  "ok": true,
  "error": 0x0000,
  "message": "成功",
  "data": { ... }
}
```

## 实现文件

- **核心路由**: `src/core/CommandRouter.cpp`
- **区域配置**: `src/core/CommandRouter_RegionConfig.cpp`
- **HTTP服务器**: `src/network/HttpServer_Command.cpp`
- **头文件**: `include/core/CommandRouter.h`

## 总结

项目共注册13个命令码，约100个命令接口（action），涵盖了系统配置、图层管理、视频播放、渲染控制、场景管理、播放列表、歌词控制、系统控制和外设控制等核心功能。

所有命令接口通过统一的命令路由系统（CommandRouter）进行分发和处理，提供了清晰的代码组织和扩展性。
