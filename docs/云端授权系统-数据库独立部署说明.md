# 云端授权系统 — 数据库独立部署说明

在**同一台 MySQL 服务器**上为「授权/设备管理系统」单独建库、建用户，与小程序数据库**完全独立**：不同库名、不同 MySQL 用户、互不访问对方数据。

---

## 1. 独立性说明

| 项目       | 小程序           | 授权/设备管理系统      |
|------------|------------------|------------------------|
| 数据库名   | 现有库（勿动）   | 新建 `license_system`  |
| MySQL 用户 | 小程序现有用户   | 新建 `license_app`     |
| 连接串     | 原配置不变       | 仅连 `license_system`  |

- 授权系统后端**只配置**连接 `license_system` 和用户 `license_app`，不接触小程序库。
- 小程序代码与配置**无需修改**，继续用原库和原用户。

---

## 2. 在服务器上执行（SSH 登录后）

### 2.1 登录 MySQL（按你当前方式，例如）

```bash
mysql -u root -p
# 或若已配置 socket/无密： mysql -u root
```

### 2.2 新建库与专用用户（复制整段执行）

```sql
-- 1) 新建数据库（仅授权系统使用）
CREATE DATABASE IF NOT EXISTS license_system
  DEFAULT CHARACTER SET utf8mb4
  DEFAULT COLLATE utf8mb4_unicode_ci;

-- 2) 新建专用用户（仅授权系统后端使用，不用于小程序）
CREATE USER IF NOT EXISTS 'license_app'@'%' IDENTIFIED BY '请改为强密码';

-- 3) 只授权该用户访问 license_system，不授权其他库
GRANT ALL PRIVILEGES ON license_system.* TO 'license_app'@'%';
FLUSH PRIVILEGES;

-- 4) 确认：查看库（应能看到 license_system，小程序的库也在列表中，不要删）
SHOW DATABASES;
```

- 将 `请改为强密码` 换成你自己的强密码，并写入授权系统后端的配置（环境变量或配置文件）。
- 若 MySQL 版本较老不支持 `CREATE USER IF NOT EXISTS`，可改为：
  ```sql
  CREATE USER 'license_app'@'%' IDENTIFIED BY '你的密码';
  GRANT ALL PRIVILEGES ON license_system.* TO 'license_app'@'%';
  FLUSH PRIVILEGES;
  ```

### 2.3 在 license_system 库中建表

```sql
USE license_system;

-- 用户（代理商 + 管理员）
CREATE TABLE IF NOT EXISTS `user` (
  `id` varchar(64) NOT NULL,
  `username` varchar(64) NOT NULL,
  `password_hash` varchar(255) NOT NULL,
  `name` varchar(128) DEFAULT NULL,
  `contact` varchar(64) DEFAULT NULL,
  `role` varchar(32) NOT NULL DEFAULT 'agent',
  `status` varchar(32) NOT NULL DEFAULT 'active',
  `created_at` bigint NOT NULL,
  `updated_at` bigint NOT NULL,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_username` (`username`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 设备
CREATE TABLE IF NOT EXISTS `device` (
  `id` varchar(64) NOT NULL,
  `serial` varchar(128) NOT NULL,
  `fingerprint` varchar(256) NOT NULL,
  `model` varchar(64) DEFAULT NULL,
  `source` varchar(32) NOT NULL DEFAULT 'manual',
  `agent_id` varchar(64) DEFAULT NULL,
  `bound_at` bigint DEFAULT NULL,
  `remark` varchar(256) DEFAULT NULL,
  `created_at` bigint NOT NULL,
  `updated_at` bigint NOT NULL,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_fingerprint` (`fingerprint`),
  KEY `idx_serial` (`serial`),
  KEY `idx_agent_id` (`agent_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 下发记录（可选）
CREATE TABLE IF NOT EXISTS `device_assign_log` (
  `id` varchar(64) NOT NULL,
  `device_id` varchar(64) NOT NULL,
  `agent_id` varchar(64) NOT NULL,
  `assigned_at` bigint NOT NULL,
  `assigned_by` varchar(64) DEFAULT NULL,
  PRIMARY KEY (`id`),
  KEY `idx_device_id` (`device_id`),
  KEY `idx_agent_id` (`agent_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

## 3. 授权系统后端连接配置

部署授权/设备管理后端时，**仅**使用新库和新用户，例如：

- **数据库名**：`license_system`
- **用户名**：`license_app`
- **密码**：上面为 `license_app` 设置的密码
- **主机/端口**：与小程序相同（本机即 `127.0.0.1` 或 `localhost`，端口一般 3306）

示例（环境变量）：

```bash
DB_HOST=127.0.0.1
DB_PORT=3306
DB_NAME=license_system
DB_USER=license_app
DB_PASSWORD=你设置的强密码
```

这样后端**只会**连接 `license_system`，不会访问小程序使用的库，实现与小程序完全独立。

## 4. 一键脚本（可选）

在服务器上保存为 `init_license_db.sh`，按需改密码后执行：`bash init_license_db.sh`（需有 MySQL 客户端且能以 root 登录）：

```bash
#!/bin/bash
MYSQL_ROOT_CMD="mysql -u root -p"   # 若 root 无密改为: mysql -u root
LICENSE_DB_PASS="请改为强密码"

$MYSQL_ROOT_CMD <<EOSQL
CREATE DATABASE IF NOT EXISTS license_system
  DEFAULT CHARACTER SET utf8mb4 DEFAULT COLLATE utf8mb4_unicode_ci;
CREATE USER IF NOT EXISTS 'license_app'@'%' IDENTIFIED BY '${LICENSE_DB_PASS}';
GRANT ALL PRIVILEGES ON license_system.* TO 'license_app'@'%';
FLUSH PRIVILEGES;
EOSQL

$MYSQL_ROOT_CMD license_system <<'EOSQL'
CREATE TABLE IF NOT EXISTS `user` (
  `id` varchar(64) NOT NULL,
  `username` varchar(64) NOT NULL,
  `password_hash` varchar(255) NOT NULL,
  `name` varchar(128) DEFAULT NULL,
  `contact` varchar(64) DEFAULT NULL,
  `role` varchar(32) NOT NULL DEFAULT 'agent',
  `status` varchar(32) NOT NULL DEFAULT 'active',
  `created_at` bigint NOT NULL,
  `updated_at` bigint NOT NULL,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_username` (`username`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE IF NOT EXISTS `device` (
  `id` varchar(64) NOT NULL,
  `serial` varchar(128) NOT NULL,
  `fingerprint` varchar(256) NOT NULL,
  `model` varchar(64) DEFAULT NULL,
  `source` varchar(32) NOT NULL DEFAULT 'manual',
  `agent_id` varchar(64) DEFAULT NULL,
  `bound_at` bigint DEFAULT NULL,
  `remark` varchar(256) DEFAULT NULL,
  `created_at` bigint NOT NULL,
  `updated_at` bigint NOT NULL,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_fingerprint` (`fingerprint`),
  KEY `idx_serial` (`serial`),
  KEY `idx_agent_id` (`agent_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE IF NOT EXISTS `device_assign_log` (
  `id` varchar(64) NOT NULL,
  `device_id` varchar(64) NOT NULL,
  `agent_id` varchar(64) NOT NULL,
  `assigned_at` bigint NOT NULL,
  `assigned_by` varchar(64) DEFAULT NULL,
  PRIMARY KEY (`id`),
  KEY `idx_device_id` (`device_id`),
  KEY `idx_agent_id` (`agent_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
EOSQL
```

## 5. 小结

- **不登录、不修改**小程序现有库和用户，只新增：
  - 库：`license_system`
  - 用户：`license_app`（仅对该库有权限）
- 授权系统后端**只连** `license_system` + `license_app`，即可保证与小程序**完全独立**。
- 你本机用 `ssh root@60.205.127.117` 免密登录后，在服务器上执行上述 SQL 或脚本即可；无需把 root 或小程序库密码交给本环境。
