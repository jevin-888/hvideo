## 📱 App版本管理功能

### 1. 进入App版本管理页面

登录后，在顶部标签栏找到 **"App版本"** 标签，点击进入。

### 2. 查看版本统计

页面顶部显示三个统计卡片：
- **总版本数**：所有版本的总数
- **已发布**：当前正在推送给用户的版本数
- **草稿**：还未发布的版本数

### 3. 查看版本列表

列表显示所有版本信息：
- 版本名称（如：1.3.0）
- 版本号（如：130）
- 文件大小
- 状态（已发布/草稿/已归档）
- 是否强制更新
- 更新说明
- 发布时间
- 创建时间
- 操作按钮

### 4. 筛选版本

- 使用状态筛选器：全部状态/已发布/草稿/已归档
- 点击"筛选"按钮应用筛选条件
- 点击"重置"按钮清除筛选

---

## ➕ 添加新版本

### 步骤1：点击"添加版本"按钮

在版本列表页面，点击右上角的 **"+ 添加版本"** 按钮。

### 步骤2：上传APK文件（推荐）

1. 点击 **"+ 添加版本"** 按钮
2. 在弹出的对话框中，点击 **"选择文件"** 按钮
3. 选择本地的APK文件（如：`hsvj-engine-v1.3.0.apk`）
4. 系统会自动：
   - ✅ 上传APK到服务器
   - ✅ 计算MD5和SHA256校验值
   - ✅ 填充下载地址、文件大小、MD5、SHA256
   - ✅ 自动提取版本号（如果文件名包含版本号）
5. 填写其他信息：
   - 版本名称：`1.3.0`（如果未自动填充）
   - 更新说明：`1. 修复了视频播放卡顿问题\n2. 优化了渲染性能`
   - 强制更新：`否`
6. 点击“保存”

### 步骤2（备选）：手动填写信息

| 字段 | 必填 | 说明 | 示例 |
|------|------|------|------|
| 版本名称 | ✅ | 显示给用户的版本名 | `1.3.0` |
| 版本号 | ✅ | 整数，必须递增 | `130` |
| 下载地址 | ✅ | APK文件的完整URL | `http://60.205.127.117:8080/downloads/hsvj-engine-v1.3.0.apk` |
| 文件大小 | ❌ | 字节单位 | `52428800`（50MB） |
| MD5校验值 | ✅ | APK文件的MD5 | `a1b2c3d4e5f6...` |
| SHA256校验值 | ❌ | APK文件的SHA256 | `abc123...` |
| 更新说明 | ❌ | 显示给用户的更新内容 | `1. 修复了视频播放卡顿问题\n2. 优化了渲染性能` |
| 强制更新 | ❌ | 是否强制用户更新 | `否` 或 `是` |
| 最低支持版本 | ❌ | 低于此版本将无法使用 | `100` |

### 步骤3：保存

点击"保存"按钮，版本将保存为**草稿**状态。

## 🚀 发布版本

### 方法1：在列表中直接发布

1. 找到状态为"草稿"的版本
2. 点击操作列的 **"发布"** 按钮
3. 确认发布

### 方法2：编辑后发布

1. 点击"编辑"按钮修改版本信息
2. 修改完成后保存
3. 点击"发布"按钮

### 发布后的效果

- ✅ 状态变为"已发布"
- ✅ App端启动时能检测到此版本
- ✅ 用户会看到更新提示对话框

---

## 📦 归档版本

### 何时需要归档

- 版本过时，不想让用户看到
- 发现版本有问题，需要撤回
- 清理旧的版本记录

### 归档操作

1. 找到状态为"已发布"的版本
2. 点击操作列的 **"归档"** 按钮
3. 确认归档

### 归档后的效果

- ✅ 状态变为"已归档"
- ❌ App端不再显示此版本
- ✅ 记录保留，可以查看历史

---

## 🗑️ 删除版本

### 可删除的版本

- ✅ 草稿状态的版本
- ✅ 已归档状态的版本
- ❌ **已发布状态的版本不能直接删除**（需要先归档）

### 删除操作

1. 找到要删除的版本
2. 点击操作列的 **"删除"** 按钮
3. 确认删除（⚠️ 不可恢复）

## 🔄 完整发布流程示例

### 场景：发布 v1.3.0 版本

#### 1. 构建APK

```bash
cd d:\Hvideo
.\scripts\build_release_apk.bat
```

#### 2. 上传APK到服务器

```bash
# 从Windows上传到服务器
scp app/build/outputs/apk/release/hsvj-engine-v1.3.0-release.apk root@60.205.127.117:/var/www/downloads/

# 或者在服务器上直接下载
ssh root@60.205.127.117
cd /var/www/downloads
wget http://your-cdn.com/hsvj-engine-v1.3.0.apk
```

#### 3. 计算MD5

```bash
# 在服务器上
md5sum /var/www/downloads/hsvj-engine-v1.3.0-release.apk
# 输出：a1b2c3d4e5f6...  hsvj-engine-v1.3.0-release.apk
```

#### 4. 查看文件大小

```bash
ls -l /var/www/downloads/hsvj-engine-v1.3.0-release.apk
# 输出：-rw-r--r-- 1 root root 52428800 ...
```

#### 5. 在管理后台添加版本

访问 `http://60.205.127.117:8080/admin`，进入"App版本"标签：

- 版本名称：`1.3.0`
- 版本号：`130`
- 下载地址：`http://60.205.127.117:8080/downloads/hsvj-engine-v1.3.0-release.apk`
- 文件大小：`52428800`
- MD5校验值：`a1b2c3d4e5f6...`（从上一步获取）
- 更新说明：`1. 修复了视频播放卡顿问题\n2. 优化了渲染性能`
- 强制更新：`否`

点击"保存"。

#### 6. 发布版本

在版本列表中找到刚添加的 v1.3.0，点击"发布"按钮。

#### 7. 测试

```bash
# 安装旧版本到测试设备
adb install -r hsvj-engine-v1.2.0.apk

# 启动App
adb shell am start -n com.hsvj.engine/.MainActivity

# 应该看到更新提示
```

## 🔧 常见问题

### Q1: 用户没有收到更新提示？

**检查清单：**
1. ✅ 版本状态是否为"已发布"
2. ✅ 版本号是否大于当前版本
3. ✅ App端能否访问下载地址
4. ✅ 查看App日志：`adb logcat | grep "更新"`

### Q2: 下载地址应该怎么写？

**格式要求：**
- 必须是完整的URL
- 确保App端能访问（内网或外网）
- 示例：
  - 外网：`https://cdn.example.com/app/hsvj-engine-v1.3.0.apk`
  - 内网：`http://60.205.127.117:8080/downloads/hsvj-engine-v1.3.0.apk`

### Q3: 如何获取APK的MD5值？

**Windows:**
```powershell
certutil -hashfile hsvj-engine-v1.3.0.apk MD5
```

**Linux/Mac:**
```bash
md5sum hsvj-engine-v1.3.0.apk
```

### Q4: 版本号规则是什么？

**规则：**
- 必须是整数
- 必须递增（130 > 129 > 128）
- 建议规则：`主版本号 * 100 + 次版本号 * 10 + 修订号`
  - v1.3.0 → 130
  - v1.2.5 → 125
  - v2.0.0 → 200

### Q5: 可以回滚版本吗？

**方法：**
1. 归档当前发布的版本
2. 重新发布旧版本（如果之前归档过）
3. 或者发布一个修正版本（版本号更高）

## 📝 版本管理最佳实践

### 1. 版本号管理

```
v1.0.0 → 100  (初始版本)
v1.1.0 → 110  (小功能更新)
v1.2.0 → 120  (功能更新)
v1.2.1 → 121  (Bug修复)
v2.0.0 → 200  (大版本更新)
```

### 2. 更新说明撰写

**好的更新说明：**
```
1. 修复了视频播放卡顿问题
2. 优化了渲染性能，提升30%
3. 新增了云端素材管理功能
4. 修复了已知的崩溃问题
```

**不好的更新说明：**
```
修复bug
```

### 3. 发布前检查

- [ ] APK已上传到服务器
- [ ] 下载链接可访问
- [ ] MD5校验值正确
- [ ] 文件大小正确
- [ ] 更新说明清晰
- [ ] 测试环境验证通过

### 4. 灰度发布策略

如果需要谨慎发布：
1. 先发布版本，但设置为小范围测试
2. 观察一段时间，确认无问题
3. 再全面推广
