# 渲染优化快速实施方案

## 方案2：GPU Instancing优化切片渲染（★★★★）

### 问题
当前每个切片都是独立的draw call：
```cpp
// 当前实现：Engine_Render.cpp:631-654
for (const RenderItem &item : renderQueue) {
    if (item.type == RenderItem::Type::SLICE) {
        renderSliceItem(item.layer, item.sliceKey, *item.sliceData);
        // 每个切片：vkMapMemory → 填充 → vkUnmapMemory → vkCmdDraw
    }
}
```

如果Layer 21有5个切片，就是5次draw call，5次GPU同步。

### 解决方案：批量渲染同一图层的所有切片

#### 实现代码

```cpp
// include/renderer/VulkanRenderer.h
class VulkanRenderer {
public:
    // 新增：批量渲染同一纹理的多个实例
    void renderLayerInstanced(
        uint32_t textureId,
        const std::vector<InstanceData>& instances
    );
    
    struct InstanceData {
        int x, y, width, height;
        float rotation, scale, alpha;
        int shapeType;
        float shapeParam;
        // ... 其他参数
    };
};

// src/core/Engine_Render.cpp
void Engine::renderLayersToCanvas(...) {
    // ... 构建renderQueue ...
    
    // 按图层分组切片
    std::map<int, std::vector<const RenderItem*>> slicesByLayer;
    for (const auto& item : renderQueue) {
        if (item.type == RenderItem::Type::SLICE) {
            slicesByLayer[item.layer->getLayerId()].push_back(&item);
        }
    }
    
    // 批量渲染
    for (const auto& [layerId, slices] : slicesByLayer) {
        if (slices.empty()) continue;
        
        Layer* layer = slices[0]->layer;
        uint32_t textureId = getLayerTextureId(layer);
        
        // 收集所有切片的instance数据
        std::vector<VulkanRenderer::InstanceData> instances;
        instances.reserve(slices.size());
        
        for (const RenderItem* item : slices) {
            VulkanRenderer::InstanceData inst;
            extractSliceInstanceData(layer, *item->sliceData, inst);
            instances.push_back(inst);
        }
        
        // 一次draw call渲染所有切片
        renderer_->renderLayerInstanced(textureId, instances);
    }
}
```

#### 预期收益
- Layer 21有5个切片：5次draw call → **1次draw call**
- GPU同步次数：5次 → **1次**
- 性能提升：**20-40%**（切片多时）

## 方案3：异步libass渲染（★★★★★）

### 问题
当前libass在主线程阻塞：
```cpp
// LyricRenderer.cpp:430-495
void LyricRenderer::prepareFrame(...) {
    // 主线程阻塞10-60ms
    ASS_Image* img = ass_render_frame(renderer_, track_, timeMs, &change);
    // ...
}
```

### 解决方案：双缓冲 + 后台线程

```cpp
// include/lyric/LyricRenderer.h
class LyricRenderer {
private:
    // 双缓冲结构
    struct LyricFrame {
        ASS_Image* assImage = nullptr;
        int64_t timestamp = -1;
        bool ready = false;
        std::mutex mutex;
    };
    
    LyricFrame frontBuffer_;  // 当前渲染使用
    LyricFrame backBuffer_;   // 后台线程准备
    
    std::thread renderThread_;
    std::atomic<bool> threadRunning_{false};
    std::atomic<int64_t> requestedTimestamp_{-1};
    
    void backgroundRenderLoop();
};

// src/lyric/LyricRenderer.cpp
void LyricRenderer::prepareFrame(double currentTime) {
    int64_t timeMs = static_cast<int64_t>(currentTime * 1000);
    
    // 异步请求：通知后台线程准备新帧
    requestedTimestamp_.store(timeMs);
    
    // 尝试切换缓冲
    {
        std::lock_guard<std::mutex> lock(backBuffer_.mutex);
        if (backBuffer_.ready && 
            std::abs(backBuffer_.timestamp - timeMs) < 100) {  // 100ms容差
            // 交换前后缓冲
            std::swap(frontBuffer_, backBuffer_);
            backBuffer_.ready = false;
        }
    }
    
    // 使用frontBuffer构建GPU mask（非阻塞）
    if (frontBuffer_.assImage) {
        buildGpuMaskFrame(frontBuffer_.assImage, currentTime);
    }
}

void LyricRenderer::backgroundRenderLoop() {
    while (threadRunning_) {
        int64_t requestedTime = requestedTimestamp_.load();
        if (requestedTime < 0) {
            std::this_thread::sleep_for(std::chrono::milliseconds(5));
            continue;
        }
        
        // 后台渲染（不阻塞主线程）
        int change = 0;
        ASS_Image* img = ass_render_frame(renderer_, track_, requestedTime, &change);
        
        if (change) {
            std::lock_guard<std::mutex> lock(backBuffer_.mutex);
            if (backBuffer_.assImage) {
                ass_free_image(backBuffer_.assImage);
            }
            backBuffer_.assImage = img;
            backBuffer_.timestamp = requestedTime;
            backBuffer_.ready = true;
        }
        
        // 重置请求
        requestedTimestamp_.compare_exchange_strong(requestedTime, -1);
    }
}
```

#### 预期收益
- 主线程阻塞：10-60ms → **0ms**
- 帧率提升：可能从45fps → **60fps**（消除卡顿）
- 延迟增加：<1帧（可接受）

#### 风险
- 需要确保libass线程安全（使用独立的ASS_Renderer实例）
- 内存增加：双缓冲（可忽略）

## 方案4：智能降频策略（★★★）

### 实现：自适应图层更新频率

```cpp
// include/layer/Layer.h
class Layer {
protected:
    int targetUpdateInterval_ = 1;  // 目标更新间隔（帧）
    int framesSinceUpdate_ = 0;
    
public:
    void setTargetFPS(int fps) {
        // 60fps系统中：30fps=每2帧，20fps=每3帧
        targetUpdateInterval_ = std::max(1, 60 / fps);
    }
    
    bool shouldUpdateThisFrame() {
        return (++framesSinceUpdate_ >= targetUpdateInterval_);
    }
    
    void markUpdated() {
        framesSinceUpdate_ = 0;
    }
};

// src/core/Engine_Render.cpp
for (const auto &layerPtr : visibleLayers) {
    Layer *layer = layerPtr.get();
    
    if (!layer->shouldUpdateThisFrame()) {
        continue;  // 跳过更新，复用上一帧纹理
    }
    
    if (layerType == LayerType::VIDEO) {
        const bool updated = videoLayer->updateVideoTexture();
        if (updated) {
            canvasNeedsRedraw = true;
            layer->markUpdated();
        }
    }
    // ...
}
```

```json
  },

  }
}
```

#### 预期收益
- 歌词从60fps → 30fps：CPU负载 **-50%**
- 静态字幕从60fps → 15fps：**-75%**

## 方案5：切片渲染缓存（★★★★）

### 问题
相同内容的切片每帧重复计算变换矩阵和渲染参数。

### 解决方案：缓存切片渲染命令

```cpp
// include/renderer/VulkanRenderer.h
class VulkanRenderer {
private:
    struct SliceRenderCache {
        uint64_t signature;  // 切片配置签名
        VkBuffer instanceBuffer;
        VkDeviceMemory instanceMemory;
        uint32_t instanceCount;
        bool valid = false;
    };
    
    std::unordered_map<std::string, SliceRenderCache> sliceCaches_;
    
public:
    void renderSliceWithCache(
        const std::string& cacheKey,
        uint32_t textureId,
        const std::vector<InstanceData>& instances
    );
};

// 使用
std::string cacheKey = std::to_string(layerId) + ":slices";
uint64_t currentSig = computeSlicesSignature(slices);

auto it = sliceCaches_.find(cacheKey);
if (it != sliceCaches_.end() && it->second.signature == currentSig) {
    // 直接使用缓存的GPU buffer
    vkCmdDrawIndexedIndirect(commandBuffer, it->second.instanceBuffer, ...);
} else {
    // 重建缓存
    rebuildSliceCache(cacheKey, currentSig, instances);
}
```

#### 预期收益
- 避免每帧重新上传instance data
- 适合静态切片配置场景
- 性能提升：**10-20%**

```cpp
// src/layer/LayerText.cpp
bool LayerText::updateTextureIfNeededForCanvas() {
```
