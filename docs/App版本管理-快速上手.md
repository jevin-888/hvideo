# App版本管理 - 快速上手指南

## 🚀 30秒发布新版本

### 操作步骤

1. **访问管理后台**
   ```
   http://60.205.127.117:8080/admin
   ```
   使用管理员账号登录

2. **进入App版本管理**
   点击顶部标签 **"App版本"**

3. **添加新版本**
   - 点击 **"+ 添加版本"** 按钮
   - 点击 **"选择文件"** 上传APK
   - 等待上传完成（自动计算MD5、SHA256，填充下载地址）
   - 填写更新说明
   - 点击 **"保存"**

4. **发布版本**
   - 在版本列表中找到刚添加的版本
   - 点击 **"发布"** 按钮
   - 确认发布

5. **完成！**
   用户下次启动App时，就能看到更新提示了！

## 📋 详细操作说明

### 上传APK（推荐方式）

**优点：**
- ✅ 自动上传文件到服务器
- ✅ 自动计算MD5和SHA256
- ✅ 自动填充下载地址和文件大小
- ✅ 自动提取版本号（从文件名）
- ✅ 显示上传进度

**操作流程：**
```
选择APK文件 → 自动上传 → 自动填充 → 填写说明 → 保存 → 发布
```

### 手动填写（备选方式）

如果不上传APK，也可以手动填写所有信息：
- 下载地址：APK的完整URL
- 文件大小：字节单位
- MD5：使用命令行工具计算
- SHA256：使用命令行工具计算

## 💡 智能功能

### 1. 自动提取版本号

如果APK文件名包含版本号，系统会自动提取：

| 文件名 | 自动提取的版本号 |
|--------|------------------|
| `hsvj-engine-v1.3.0.apk` | `1.3.0` |
| `app-v2.0.1-release.apk` | `2.0.1` |
| `myapp_v1.2.3.apk` | `1.2.3` |

### 2. 自动计算校验值

上传APK后，系统会自动计算：
- **MD5**：用于文件完整性校验
- **SHA256**：更安全的校验算法

### 3. 实时上传进度

上传大文件时，显示实时进度条：
```
上传中... 45%
```

## 🔧 命令行计算MD5（如果手动填写）

### Windows
```powershell
certutil -hashfile hsvj-engine-v1.3.0.apk MD5
certutil -hashfile hsvj-engine-v1.3.0.apk SHA256
```

### Linux/Mac
```bash
md5sum hsvj-engine-v1.3.0.apk
sha256sum hsvj-engine-v1.3.0.apk
```

## ⚠️ 注意事项

### 1. 文件大小限制
- 最大支持：500MB
- 如果APK超过500MB，需要手动上传到服务器

### 2. 版本号规则
- 必须是整数
- 必须递增（130 > 129 > 128）
- 建议：`主版本*100 + 次版本*10 + 修订号`
  - v1.3.0 → 130
  - v2.0.1 → 201

### 3. 下载地址格式
- 必须是完整URL
- 确保App端可以访问
- 示例：`http://60.205.127.117:8080/downloads/hsvj-engine-v1.3.0.apk`

### 4. 更新说明撰写
**好的示例：**
```
1. 修复了视频播放卡顿问题
2. 优化了渲染性能，提升30%
3. 新增了云端素材管理功能
```

**不好的示例：**
```
修复bug
```

## 🎯 完整示例

### 场景：发布 v1.3.0

**准备：**
- APK文件：`hsvj-engine-v1.3.0.apk`
- 更新内容：
  1. 修复了视频播放卡顿问题
  2. 优化了渲染性能

**操作：**

1. 登录 `http://60.205.127.117:8080/admin`

2. 点击 **"App版本"** 标签

3. 点击 **"+ 添加版本"**

4. 点击 **"选择文件"**，选择 `hsvj-engine-v1.3.0.apk`

5. 等待上传完成（约10-30秒，取决于网络速度）

6. 系统自动填充：
   - 下载地址：`http://60.205.127.117:8080/downloads/1714132800000_hsvj-engine-v1.3.0.apk`
   - 文件大小：`52428800`
   - MD5：`a1b2c3d4e5f6...`
   - SHA256：`abc123...`
   - 版本名称：`1.3.0`（自动提取）

7. 手动填写：
   - 版本号：`130`
   - 更新说明：`1. 修复了视频播放卡顿问题\n2. 优化了渲染性能`
   - 强制更新：`否`

8. 点击 **"保存"**

9. 在版本列表中点击 **"发布"**

10. 确认发布

**完成！** 🎉

## 🔍 问题排查

### Q1: 上传失败？
**检查：**
- 网络连接是否正常
- 文件大小是否超过500MB
- 是否为.apk文件

### Q2: 发布后用户看不到更新？
**检查：**
- 版本状态是否为"已发布"
- 版本号是否大于用户当前版本
- App端能否访问下载地址

### Q3: 如何修改已发布的版本？
**方法：**
1. 归档当前版本
2. 创建新版本（版本号更高）
3. 发布新版本
