## 🔍 核心问题诊断

### 问题1：超4K视频自动播放（主要原因）

**证据**：
```
[VideoLayerDiag] layer=1 state=PLAYING visible=1
video=2626x2160 
path=/huoshan/video/人像视频2625x2160.mp4
```

**影响**：
- 占用大量GPU带宽
- 软件解码占用CPU
- 导致帧时间从<16ms增加到51ms
- 帧率从60fps降至19fps

### 问题2：采集重新初始化导致527ms阻塞

**证据**：
```
stage=dma_buf_import cost=527ms (首次导入)
stage=ycbcr_pipeline_warmup cost=1222ms (管线预热)
```

**原因**：
- 你移动采集图层时，采集设备被重新打开
- 首次DMA导入需要分配GPU内存（527ms）
- 首次YCbCr管线创建需要编译着色器（1222ms）
- **这些是一次性开销，不是持续的性能问题**

### 问题3：我的优化状态

**关键发现**：
```bash
grep "dma_buf_rebind" 日志
# 结果：无输出
```

**说明**：
- **没有发生dma_buf_rebind操作**
- 采集图层重新初始化后，创建了全新的纹理
- 我的优化（fence替代waitIdle）针对的是"更新现有纹理"的场景
- 当前是"首次创建纹理"，不会触发rebind

**结论**：我的优化代码没问题，但当前场景不适用。
