### 3.1 Network API 边界

- `src/network/HttpServer_*.cpp`：公开 HTTP API，唯一前缀为 `/api/v1`。
- `src/network/HttpServer_Command.cpp`：把 `POST /api/v1/{module}/actions/{action}` 精准映射到一个模块和一个 action；请求 body 仅含业务参数。
- `src/core/CommandRouter_*.cpp`：仅作为进程内应用层执行器，不暴露通用命令 HTTP 入口。
- `src/network/web/shared/moduleActions.js`：前端模块 action 的唯一请求构造器，拒绝 `type/code/param/action` 顶层字段。
- `src/network/web/shared/apiResponseParser.js`：前端唯一响应 parser，只接受 `{ok,data,error}`。
- `scripts/check_api_contract.py`：静态检查 API 前缀、重复路由、禁用入口、模块注册和旧 wire 字段。

该项目尚未发布，因此不保留任何旧 HTTP 协议兼容层。播放列表资源统一使用复数 `/api/v1/playlists`；外设 action 统一使用 `peripheral_type/template_code/data` 等业务字段。

## 10. 第三方依赖模块

`vendor/` 中包含当前工程直接或间接依赖的源码/预构建库：`ffmpeg-rockchip`、`freetype-2.8`、`fribidi`、`harfbuzz`、`jsoncpp`、`libass`、`libdrm-android`、`LibJpeg-Turbo-Android-master`、`libpng`、`libyuv`、`openssl-android`、`qrcodegen`、`rkmpp`、`security8.1`、`security8.2`、`sqlcipher`、`sqlite`、`sqlite3mc`、`stb`、`WifiDisplay_backup`、`zlib`。

`Third-Party/` 中还保留大型历史依赖、素材和工具集合。清理时需要先确认是否仍被 CMake、Gradle、脚本或发布包引用。

## 11. 清理审核原则

- 先审业务源码和构建入口，再处理资源和脚本，最后处理第三方和历史目录。
- 不把 `target`、`build`、`.gradle`、`node_modules`、IDE 配置、压缩包、日志和缓存视为业务模块。
- 删除代码前必须确认没有被构建脚本、JNI、反射、路由、静态资源或外部配置引用。
- 对无法确认的历史目录先标注为“待归档/待确认”，不直接删除。
- 新项目未发布，业务代码中不保留临时占位、未接线样例、明显不可达分支和调试输出。
